About Workera ELO

How the platform works before you write a single answer

📐 What is Workera ELO?

Workera is an AI-powered skills assessment platform (founded by Andrew Ng's team via DeepLearning.AI) that measures your ML/AI skills using a Computerized Adaptive Test (CAT) powered by an Elo rating algorithm. Your score is not fixed — it dynamically updates after every question based on both your answer and the item's current difficulty rating.

Skills are measured on a 1–300 scale: Beginner (1–100), Developing (101–200), Accomplished (201–300). The platform starts you with a mid-difficulty question and moves up or down based on correct/incorrect answers, converging on your true ability level in ~20 questions per domain.

ELO Scoring Mechanics

How the Elo Update Works

Each question has a difficulty rating D and your skill has a rating S. The expected probability of a correct answer is:

E(correct) = 1 / (1 + 10^((D - S) / 400)) After answer: S_new = S + K × (actual − expected) actual = 1 (correct), 0 (incorrect) K = sensitivity factor (typically 16–32)

Adaptive Difficulty Branching

After each answer the algorithm selects the next question whose difficulty best discriminates at your current estimated skill level (roughly ±1 standard deviation). This means 20 questions carry more statistical power than a 60-question fixed test. Speed matters marginally: faster correct responses yield slightly larger positive updates; slower responses converge toward zero delta even if correct.

Implementing Linear Regression

Track A · Applying Foundational Techniques
1.1 — Gradient Descent Implementation

Core Math You Must Know Cold

Model: ŷ = Xθ, where X ∈ ℝ^(m×(n+1)) with bias column prepended. Cost: J(θ) = (1/2m)‖Xθ − y‖². Gradient: ∇J = (1/m)Xᵀ(Xθ − y). Update rule: θ ← θ − α·∇J. Vectorized update handles all parameters simultaneously. Batch GD uses full dataset per step; SGD uses one sample; mini-batch uses a chunk (typical 32–512).

MCQ Medium
You run gradient descent on a linear regression problem and observe that the cost oscillates wildly and diverges after several iterations. The learning rate α = 1.5. What is the most likely cause and what is the appropriate fix?
AThe features are not normalized — apply z-score standardization.
BThe learning rate is too large — the gradient update overshoots the minimum; reduce α.
CThe model is underfitting — add polynomial features to increase capacity.
DThe cost function is non-convex — switch to a different optimizer.
Explanation & Key Insight
Answer: B. Linear regression has a convex quadratic cost surface. Diverging oscillations are the classic symptom of α exceeding the critical threshold α < 2/λ_max where λ_max is the largest eigenvalue of (1/m)XᵀX. At α=1.5 the parameter vector alternates between sides of the bowl and grows. Fix: reduce α (try 0.01, 0.1) and monitor the cost curve — it should decrease monotonically. Feature normalization (A) helps GD converge faster but does not cause divergence on its own.
💡 On Workera, divergence questions often follow up by asking you to identify the learning curve shape. Divergence = J increases. Slow convergence = J decreases but very slowly. Oscillation without divergence = α slightly too large but below threshold.
Coding
Implement vectorized batch gradient descent for linear regression in Python/NumPy. Your function should return the learned parameter vector theta and the cost history array. Do not use sklearn.
Reference Implementation
Python · NumPy
import numpy as np

def gradient_descent_linreg(
    X: np.ndarray,          # (m, n) feature matrix — NO bias column yet
    y: np.ndarray,          # (m,) target vector
    alpha: float = 0.01,
    num_iters: int = 1000
) -> tuple[np.ndarray, np.ndarray]:

    m = X.shape[0]

    # Prepend bias column of ones
    X_b = np.column_stack([np.ones(m), X])   # (m, n+1)
    n1  = X_b.shape[1]
    theta = np.zeros(n1)                     # parameter init
    cost_history = np.zeros(num_iters)

    for i in range(num_iters):
        residual = X_b @ theta - y           # (m,)   ŷ − y
        cost_history[i] = (1 / (2 * m)) * residual @ residual
        gradient = (1 / m) * X_b.T @ residual  # (n+1,)
        theta -= alpha * gradient            # simultaneous update

    return theta, cost_history


# ── Quick sanity check ──────────────────────────────
if __name__ == "__main__":
    rng  = np.random.default_rng(42)
    X    = rng.normal(size=(200, 2))
    true = np.array([3.0, -1.5, 2.0])   # bias, w1, w2
    y    = np.column_stack([np.ones(200), X]) @ true + rng.normal(scale=0.3, size=200)

    theta, history = gradient_descent_linreg(X, y, alpha=0.05, num_iters=2000)
    print(f"Learned θ: {theta}")            # Should be ≈ [3, -1.5, 2]
    print(f"Final cost: {history[-1]:.4f}")
Critical details Workera tests:
  • Prepend bias column before the loop (not inside) — efficiency matters.
  • Vectorized gradient = (1/m) Xᵀ(Xθ−y), not a Python for-loop over samples.
  • Update is simultaneous: compute full gradient first, then update θ.
  • Cost formula uses 1/(2m) so that the gradient has a clean 1/m coefficient.

MCQ Medium
You're predicting house prices. Your dataset has a few extreme outlier sales (10× median price). Which metric is most robust to these outliers when evaluating model quality?
AMSE — mean squared error punishes all errors proportionally.
BMAE — mean absolute error gives equal weight to all deviations regardless of magnitude.
CR² — coefficient of determination is always between 0 and 1.
DRMSE — root mean squared error corrects the scale issue of MSE.
Explanation
Answer: B. MAE grows linearly with error size so a 10× outlier contributes exactly 10× the loss of a typical error. MSE grows quadratically — that same outlier contributes 100× the loss, dominating the metric and misleading you about typical-case performance. RMSE is the square root of MSE so still outlier-sensitive. R² can even go negative when the model is terrible, and is distorted by outliers via SStot.
Subjective Hard
A model achieves R² = 0.95 on training data and R² = 0.40 on test data. Another model achieves R² = 0.75 on training and R² = 0.72 on test. Compare these models. Which would you deploy and why?
Model Answer
Model A (0.95 train / 0.40 test) is heavily overfitted: it memorizes training patterns but generalizes poorly. The 0.55 train-test gap indicates high variance. Model B (0.75 / 0.72) shows a small, acceptable generalization gap (0.03), demonstrating stability and genuine pattern learning.

Deploy Model B. In production, test R² predicts real-world performance. Model A's high training R² is misleading — it would fail on new data. Remedies for Model A would include regularization (Ridge/Lasso), dropout (if neural), or reduced model complexity.
💡 Workera often pairs this with a follow-up asking what regularization technique to apply. Know that Ridge (L2) shrinks coefficients toward zero; Lasso (L1) produces sparsity; ElasticNet combines both.
1.3 — Data Preprocessing for Regression
MCQ Medium
Feature matrix X has columns with very different scales: age ∈ [18,80] and salary ∈ [30000,500000]. You apply z-score standardization. Which of the following best describes its effect on gradient descent for linear regression?
AIt changes the optimal θ* values, leading to a different final model.
BIt has no effect; gradient descent is scale-invariant by definition.
CIt makes the cost surface more spherical, allowing a uniform learning rate to converge faster without changing the predictions.
DIt prevents overfitting by shrinking the parameter values toward zero.
Explanation
Answer: C. Without normalization the cost bowl is elongated along the low-scale axis (age) because tiny steps in salary direction cause huge cost changes. Gradient descent zig-zags slowly. After standardization (μ=0, σ=1 per feature), the bowl is approximately spherical, one α works for all parameters, and convergence is dramatically faster. The final predictions are mathematically equivalent once you back-transform θ. This is distinct from regularization (D).
Coding Medium
Write a preprocessing pipeline (without sklearn) that: (1) imputes numeric NaNs with column medians, (2) standardizes all features, (3) returns both the transformed X and a transform function that can be applied to test data using only training statistics.
Reference Implementation
Python · NumPy
import numpy as np

def fit_transform(X_train: np.ndarray):
    """
    Returns (X_scaled, transform_fn) where transform_fn applies
    train-fitted statistics to new data — preventing data leakage.
    """
    # Step 1: Compute column medians ignoring NaN
    medians = np.nanmedian(X_train, axis=0)  # shape (n,)

    # Step 2: Fill NaN with median (copy to avoid mutation)
    X_imp = X_train.copy().astype(float)
    for j in range(X_imp.shape[1]):
        mask = np.isnan(X_imp[:, j])
        X_imp[mask, j] = medians[j]

    # Step 3: Fit scaler on imputed train data
    mu    = X_imp.mean(axis=0)    # shape (n,)
    sigma = X_imp.std(axis=0)     # shape (n,)
    sigma[sigma == 0] = 1.0      # avoid division by zero for const cols

    X_scaled = (X_imp - mu) / sigma

    def transform(X_new: np.ndarray) -> np.ndarray:
        X_c = X_new.copy().astype(float)
        for j in range(X_c.shape[1]):
            mask = np.isnan(X_c[:, j])
            X_c[mask, j] = medians[j]  # use TRAIN median
        return (X_c - mu) / sigma      # use TRAIN mu / sigma

    return X_scaled, transform


# ── Usage ────────────────────────────────────────────
X_tr = np.array([[1.0, np.nan], [3.0, 2.0], [np.nan, 4.0], [5.0, 6.0]])
X_te = np.array([[2.0, np.nan], [4.0, 1.0]])

X_tr_s, tfn = fit_transform(X_tr)
X_te_s      = tfn(X_te)   # no leakage — uses train stats only
💡 Data leakage is a frequent Workera trap question. NEVER compute mean/std on the full dataset before splitting. Always fit the scaler on train only, then apply to val/test.

Implementing Logistic Regression

Track A · Binary Classification
2.1 — Classification Performance Metrics

Confusion Matrix & Derived Metrics

TP = true positives, TN = true negatives, FP = false positives (Type I), FN = false negatives (Type II). All classification metrics derive from this 2×2 table.

Accuracy = (TP + TN) / (TP + TN + FP + FN) Precision = TP / (TP + FP) ← "of predicted positives, how many are real?" Recall = TP / (TP + FN) ← "of real positives, how many did we catch?" F1 = 2 × (P × R) / (P + R) ← harmonic mean; balanced P–R tradeoff Specificity = TN / (TN + FP) AUC-ROC = area under TPR vs FPR curve ∈ [0.5, 1.0]
MCQ Hard
You're building a cancer detection classifier. Missing a real positive (cancer patient classified as healthy) is far more dangerous than a false alarm (healthy patient flagged for more tests). Which metric should you maximize?
APrecision — minimizes the number of false alarms sent to patients.
BRecall (Sensitivity) — minimizes the number of missed cancer cases.
CAccuracy — gives the overall fraction of correct classifications.
DSpecificity — measures how well we identify healthy patients correctly.
Explanation
Answer: B. Recall = TP/(TP+FN) penalizes FN (missed cancers). Maximizing recall means we lower the classification threshold, accepting more FP (unnecessary follow-ups) but reducing FN (missed cases). Precision (A) would have the opposite effect. In healthcare and fraud detection, the cost asymmetry between FN and FP is explicit — always ask: "which error costs more?" before picking a metric.
💡 Workera asks this via scenario framing. Map: FN-costly → maximize Recall; FP-costly → maximize Precision; balanced → F1; threshold-agnostic → AUC-ROC.
MCQ Medium
A dataset has 950 negative and 50 positive samples. A naive classifier that always predicts negative achieves accuracy = 95%. What metric would best expose this model's failure?
AMSE of the predicted probabilities.
BF1-score or AUC-ROC — both penalize the zero-recall on the minority class.
CLog-loss (binary cross-entropy).
DAccuracy on the negative class only.
Explanation
Answer: B. The naive classifier has Recall = 0 (no positives predicted) and Precision = undefined (0/0). F1 = 0. AUC-ROC = 0.5 (random). Both immediately reveal the failure. Accuracy (95%) is misleading for imbalanced data — this is a well-known pitfall. Log-loss (C) would give a finite but misleading value because predicted probability for negatives is 1.0.
2.2 — Decision Boundary & Output Interpretation

Sigmoid & Decision Boundary

Logistic regression outputs P(y=1|x) = σ(θᵀx) = 1/(1+e^(−θᵀx)). Decision boundary is the set {x : θᵀx = 0} — a hyperplane in feature space. The boundary is linear in feature space but becomes curved when polynomial features are added.

MCQ Medium
A logistic regression model outputs P(y=1|x) = 0.3. Using a threshold of 0.5, the prediction is class 0. You lower the threshold to 0.25. How does this change the model's behavior across the dataset?
AMore samples are classified as class 0; recall for class 1 decreases.
BMore samples are classified as class 1; recall for class 1 increases but precision for class 1 likely decreases.
CThe model probabilities change but the classification boundary stays the same.
DThe model is retrained with a new decision boundary during optimization.
Explanation
Answer: B. Lowering the threshold moves the decision boundary so fewer raw probability is needed to call class 1. More points cross into the positive region → more TP and more FP → recall increases, precision decreases (P–R tradeoff). The model probabilities are fixed (no retraining) — only the post-hoc threshold changes. This is the foundation for ROC curve construction.
Subjective Hard
Explain why logistic regression output probabilities are not necessarily calibrated, and describe one technique to calibrate them.
Model Answer
Logistic regression is trained to maximize log-likelihood, not to minimize calibration error. On an imbalanced dataset or when decision boundaries are near dense data regions, the sigmoid outputs can be systematically over- or under-confident (e.g., outputting 0.9 for a sample that historically is positive only 60% of the time).

Calibration techniques:
  • Platt Scaling: fit a second logistic regression on the model's raw scores using a hold-out set. Cheap and effective for sigmoid-shaped reliability diagrams.
  • Isotonic Regression: non-parametric, monotone transformation fitted on the calibration set — more flexible than Platt, but requires more data.
Evaluate calibration with a reliability diagram (plot mean predicted probability vs actual fraction in each bin) and the Expected Calibration Error (ECE).
2.3 — Binary Classification Implementation
Coding Hard
Implement logistic regression from scratch: sigmoid, binary cross-entropy loss, gradient computation, and a predict method. Then train it on a toy XOR-like dataset and explain why the linear model fails.
Reference Implementation
Python · NumPy
import numpy as np

class LogisticRegression:
    def __init__(self, alpha=0.1, n_iter=1000, lam=0.0):
        self.alpha, self.n_iter, self.lam = alpha, n_iter, lam

    @staticmethod
    def _sigmoid(z): return 1 / (1 + np.exp(-np.clip(z, -500, 500)))

    def fit(self, X, y):
        m, n = X.shape
        X_b  = np.column_stack([np.ones(m), X])
        self.theta = np.zeros(n + 1)

        for _ in range(self.n_iter):
            p       = self._sigmoid(X_b @ self.theta)  # (m,)
            # Binary cross-entropy gradient
            grad    = (1/m) * X_b.T @ (p - y)
            # L2 regularization (skip bias term)
            reg     = np.concatenate([[0], self.lam * self.theta[1:]])
            self.theta -= self.alpha * (grad + reg)
        return self

    def predict_proba(self, X):
        X_b = np.column_stack([np.ones(len(X)), X])
        return self._sigmoid(X_b @ self.theta)

    def predict(self, X, threshold=0.5):
        return (self.predict_proba(X) >= threshold).astype(int)


# XOR dataset — logistic regression cannot separate this
X_xor = np.array([[0,0],[0,1],[1,0],[1,1]])
y_xor = np.array([0,1,1,0])

model = LogisticRegression(alpha=0.5, n_iter=5000).fit(X_xor, y_xor)
print(model.predict(X_xor))  # Will NOT achieve [0,1,1,0] — XOR is not linearly separable

# Fix: add polynomial feature x1*x2
X_poly = np.column_stack([X_xor, X_xor[:,0] * X_xor[:,1]])
model2 = LogisticRegression(alpha=0.5, n_iter=10000).fit(X_poly, y_xor)
print(model2.predict(X_poly))  # Now converges to [0,1,1,0]
Why XOR fails without feature engineering: XOR requires a non-linear decision boundary. Logistic regression can only learn a hyperplane in feature space. Adding the interaction term x₁×x₂ makes the problem linearly separable in the augmented space.

MCQ Medium
You're designing a neural network for 10-class image classification. The final hidden layer has 128 units. Which output configuration is correct?
A1 output neuron with sigmoid activation; threshold at 0.5 per class.
B10 output neurons with ReLU activation; argmax gives the class.
C10 output neurons with Softmax activation; outputs sum to 1 and represent class probabilities.
D10 output neurons with Sigmoid activation paired with binary cross-entropy.
Explanation
Answer: C. Softmax enforces mutual exclusivity (outputs sum to 1), appropriate when exactly one class is correct. ReLU (B) can output negatives and doesn't normalize — invalid for probabilities. Sigmoid per neuron (D) is for multi-label where multiple classes can be simultaneously true. The correct loss is categorical cross-entropy (sparse or one-hot form).
MCQ Hard
A deep network (20 hidden layers) with sigmoid activations throughout fails to learn — training loss barely decreases from initialization. What is the most likely cause?
AThe learning rate is too high, causing gradient explosion.
BThe network is too wide — reduce the number of neurons per layer.
CVanishing gradients: sigmoid's derivative ≤ 0.25 compresses gradients to near-zero after repeated multiplication through layers.
DOverfitting — add more training data or dropout.
Explanation
Answer: C. σ'(z) = σ(z)(1−σ(z)) ≤ 0.25 always. Backprop multiplies gradients through L layers: 0.25^20 ≈ 10⁻¹² — essentially zero. Early layers receive no gradient signal. Fix: replace sigmoid with ReLU (or variants: Leaky ReLU, ELU, GELU) which have gradient = 1 for positive inputs. Also use He initialization, batch normalization, and residual connections for very deep nets.
💡 Memorize: ReLU derivative = 1 (z>0), 0 (z≤0). Leaky ReLU = 1 (z>0), α≈0.01 (z≤0). ELU = 1 (z>0), αeᶻ (z≤0). GELU is used in Transformers.
3.2 — Training & Hyperparameter Tuning

Key Hyperparameters to Know

Learning rate (most impactful), batch size, depth, width, activation functions, optimizer (SGD, Adam, RMSProp), regularization (L1/L2, dropout rate), number of epochs, weight initialization strategy.

MCQ Medium
The Adam optimizer maintains two exponential moving averages of gradients. What do they represent and why does this help?
APast gradients and past loss values — reduces overfitting.
BFirst moment (mean of gradients) and second moment (uncentered variance) — enables per-parameter adaptive learning rates with momentum.
CGradient and Hessian — approximates second-order optimization.
DCurrent gradient and previous gradient — computes a velocity term only.
Explanation
Answer: B. Adam: m_t = β₁m_{t-1} + (1−β₁)g_t (first moment), v_t = β₂v_{t-1} + (1−β₂)g_t² (second moment). Bias-corrected: m̂_t = m_t/(1−β₁ᵗ), v̂_t = v_t/(1−β₂ᵗ). Update: θ ← θ − α m̂_t / (√v̂_t + ε). Parameters with consistently large gradients get smaller effective learning rates (v̂_t large); sparse parameters get larger effective rates. Defaults: β₁=0.9, β₂=0.999, ε=10⁻⁸.
Subjective Medium
Describe the role of early stopping and dropout as regularization techniques for neural networks. Are they complementary or redundant?
Model Answer
Early stopping monitors validation loss and halts training when it starts increasing (indicating overfitting). It's equivalent to L2 regularization in some regimes — it prevents the model from memorizing training noise by limiting effective model complexity through iteration count.

Dropout randomly zeros out neurons (probability p, typically 0.2–0.5) during each forward pass in training. This forces the network to learn redundant representations and prevents co-adaptation of neurons, analogous to training an ensemble of thinned networks.

They are complementary: dropout controls model capacity per step; early stopping controls when to stop. Both reduce overfitting through different mechanisms and are typically used together. Note: dropout is disabled at inference time (scale outputs by 1−p or use inverted dropout).
3.3 — Neural Networks with Standard Frameworks
Coding Medium
Build a 3-layer feedforward neural network in PyTorch for binary classification (input dim=20). Include a training loop with Adam optimizer and evaluate on a held-out set.
Reference Implementation
PyTorch
import torch, torch.nn as nn
from torch.utils.data import DataLoader, TensorDataset

class BinaryMLP(nn.Module):
    def __init__(self, input_dim=20):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(input_dim, 64),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(64, 32),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(32, 1),
            nn.Sigmoid()
        )
    def forward(self, x): return self.net(x).squeeze(-1)


def train(X_tr, y_tr, X_val, y_val, epochs=50, batch=64):
    model     = BinaryMLP()
    optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
    criterion = nn.BCELoss()

    loader = DataLoader(TensorDataset(X_tr, y_tr), batch_size=batch, shuffle=True)

    for ep in range(epochs):
        model.train()
        for xb, yb in loader:
            optimizer.zero_grad()
            loss = criterion(model(xb), yb)
            loss.backward()
            optimizer.step()

        # Validation
        model.eval()
        with torch.no_grad():
            val_preds = (model(X_val) >= 0.5).float()
            acc = (val_preds == y_val).float().mean().item()
        if (ep+1) % 10 == 0:
            print(f"Epoch {ep+1:3d} | val_acc={acc:.3f}")
    return model


# Toy data
X = torch.randn(1000, 20); y = (X[:, 0] > 0).float()
split = 800
trained = train(X[:split], y[:split], X[split:], y[split:])
Workera tests: BCELoss vs BCEWithLogitsLoss (latter is numerically more stable — combine logits + loss), zero_grad placement, model.eval() + no_grad() during validation.

Decision Trees & Ensemble Methods

Track B · Developing Intermediate Models
4.1 — Building & Interpreting Decision Trees

Splitting Criteria

Gini Impurity: G = 1 − Σ pᵢ². Measures probability that a randomly chosen sample is misclassified if labelled according to class distribution. Entropy: H = −Σ pᵢ log₂(pᵢ). Information-theoretic measure of impurity. Information Gain: IG = H(parent) − Σ(|child|/|parent|)H(child). Both give similar splits empirically; Gini is computationally cheaper.

MCQ Medium
A leaf node in a decision tree has class distribution: 8 samples of class A, 2 samples of class B. What is the Gini impurity of this node?
A0.5
B0.32
C0.20
D0.72
Calculation
Answer: B. p_A = 8/10 = 0.8, p_B = 2/10 = 0.2. G = 1 − (0.8² + 0.2²) = 1 − (0.64 + 0.04) = 1 − 0.68 = 0.32. A pure node (all one class) has G=0; maximum impurity for 2 classes is G=0.5 (50/50 split). G=0.32 indicates a fairly pure node.
Subjective Medium
What is the difference between pre-pruning and post-pruning in decision trees? Describe one hyperparameter for each.
Model Answer
Pre-pruning (early stopping) restricts tree growth during construction by halting splits when conditions are met. Key hyperparameters: max_depth (limits tree height), min_samples_split (minimum samples required to split a node), min_impurity_decrease (split only if impurity drops by at least this amount).

Post-pruning grows a full tree first, then removes branches that provide insufficient generalization benefit on a validation set. Key technique: cost-complexity pruning (ccp_alpha in sklearn) adds a penalty α × (number of leaves) to the cost — higher α prunes more aggressively.

Post-pruning generally produces better trees because it has the full context of the tree before deciding what to remove, but it is more computationally expensive.

MCQ Hard
Bagging reduces variance but not bias. Boosting reduces bias but can increase variance. Given a model that underfits (high bias, low variance), which ensemble technique is more appropriate?
ABagging with many deep trees — reduces variance of the underfitting base learners.
BBoosting with weak learners — sequentially corrects errors and reduces bias.
CNeither — ensembles cannot fix high bias; add more features instead.
DBagging with shallow stumps — depth controls variance independently of bias.
Explanation
Answer: B. Bagging averages parallel models to cancel out independent errors (variance reduction). If each base learner already has high bias, averaging them doesn't fix it — you average high-bias models and still get high bias. Boosting trains sequentially, each model specializing in the errors of its predecessors, directly reducing the combined model's bias at the cost of some added variance. The classic framing: Bagging for overfitting; Boosting for underfitting.

Train a Random Forest classifier with sklearn and extract + plot feature importances. Explain what "mean decrease in impurity" (MDI) means and name one limitation.

from sklearn.datasets import make_classification

                            random_state=42)

                             random_state=42).fit(X, y)

    print(f"Rank {rank+1}: feature_{idx} | importance = {importances[idx]:.4f}")
MDI (Mean Decrease in Impurity): for each feature, sum the weighted Gini/entropy reduction across all splits on that feature across all trees, normalized to sum to 1. Higher MDI = more useful feature for splitting.

Limitation: MDI is biased toward high-cardinality features (continuous variables or categorical variables with many unique values) because they offer more possible split points, artificially inflating their apparent importance. Use permutation importance on a hold-out set for an unbiased alternative.
4.3 — Bias–Variance Tradeoff
Classification Hard
Classify each scenario as high bias, high variance, well-balanced, or high noise:

(a) Train MSE = 0.05, Val MSE = 0.06
(b) Train MSE = 0.08, Val MSE = 4.2
(c) Train MSE = 3.5, Val MSE = 3.7
(d) Train MSE = 0.01, Val MSE = 0.01 but both are above the irreducible error floor of 3.0
Full Classification
(a) 0.05 / 0.06 → Well-balanced. Low train error, very small generalization gap.
(b) 0.08 / 4.2 → High variance (overfitting). Model memorizes training data, fails on validation. Fix: regularize, more data, simpler model.
(c) 3.5 / 3.7 → High bias (underfitting). Both errors are high; gap is small. Model is too simple. Fix: add features, increase model complexity, longer training.
(d) 0.01 / 0.01 but floor = 3.0 → Impossible / data leakage or test contamination. Errors below irreducible noise floor indicate a data leak (train and test share labels or identical rows).
💡 The bias-variance decomposition: E[(y−ŷ)²] = Bias² + Variance + Irreducible Noise. You can never beat the noise floor regardless of model complexity.

MCQ Medium
K-Means clustering is applied to a dataset with two crescent-shaped clusters. The algorithm consistently assigns points incorrectly. What is the fundamental reason?
AK was set too high, causing over-segmentation.
BK-Means converged to a local minimum due to poor centroid initialization.

DK-Means requires normalized features; the dataset was not standardized.
Explanation
Answer: C. K-Means minimizes within-cluster sum of squared Euclidean distances. Its Voronoi-partition decision boundaries are always linear (straight hyperplanes) in feature space — they literally cannot capture crescent, ring, or interleaved cluster shapes. Fix: use DBSCAN (density-based, no shape assumption), or apply kernel K-Means / spectral clustering which implicitly maps to a higher-dimensional space where crescents become separable.

Implement the elbow method to find the optimal K for K-Means clustering. Also compute the silhouette score for each K and explain what it measures.
Reference Implementation
Python · sklearn
from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score
from sklearn.datasets import make_blobs
import numpy as np

X, _ = make_blobs(n_samples=500, centers=4, random_state=42)

inertias, sil_scores, K_range = [], [], range(2, 11)

for k in K_range:
    km     = KMeans(n_clusters=k, init="k-means++", n_init=10, random_state=42)
    labels = km.fit_predict(X)

    inertias.append(km.inertia_)               # WCSS for elbow plot
    sil_scores.append(silhouette_score(X, labels))  # higher = better

# Find elbow: largest second-difference in inertia
deltas  = np.diff(inertias)
d2      = np.diff(deltas)
elbow_k = K_range.start + np.argmin(d2) + 1  # +1 for 0-indexing offset
best_sil_k = K_range.start + np.argmax(sil_scores)

print(f"Elbow K estimate: {elbow_k}")
print(f"Best silhouette K: {best_sil_k} (score={max(sil_scores):.3f})")
Silhouette score s(i) = (b(i) − a(i)) / max(a(i), b(i)) where a(i) = mean intra-cluster distance and b(i) = mean distance to nearest other cluster. Score ∈ [−1, 1]: close to 1 = well-clustered; close to 0 = on boundary; negative = possibly wrong cluster.
5.2 — Anomaly Detection

Key Methods

Statistical: Z-score, IQR (for univariate). Density-based: Isolation Forest, Local Outlier Factor (LOF). Reconstruction-based: Autoencoder (high reconstruction error → anomaly). Distance-based: k-NN distance. Choice depends on dimensionality, interpretability needs, and whether you need a contamination estimate.

MCQ Hard
Isolation Forest detects anomalies by building random trees. Which statement best explains why anomalies are isolated with fewer splits?
AAnomalies have extreme feature values that cause the tree to grow imbalanced toward them.
BAnomalies are sparse and far from other points; random axis-aligned splits isolate them quickly in few steps. Normal points are dense and require many splits to separate.
CIsolation Forest uses the training error signal to penalize anomalous points harder.
DAnomalies have lower feature variance, making them easier to partition.
Explanation
Answer: B. Isolation Forest's key insight: anomalies lie in low-density regions. A random split on any feature axis is likely to land between an anomaly and the bulk of data quickly. In contrast, a normal point surrounded by neighbors requires many recursive splits before it's isolated in its own leaf. Average path length ≈ log(n) for normal points; much shorter for anomalies. Anomaly score = 1 − E[path length] / c(n) where c(n) is the average path length of an unsuccessful BST search.
Subjective Hard
In anomaly detection, labels are almost never available. Describe how you would choose a contamination threshold (the fraction of training data treated as anomalies) in an unsupervised setting.
Model Answer
Several strategies exist:
  • Domain prior: Use known or estimated base rate. E.g., fraud is ~0.1% of transactions — set contamination=0.001.
  • Score distribution inspection: Plot the histogram of anomaly scores. A bimodal distribution suggests a natural threshold at the valley between modes.
  • ROC / PR curve with partial labels: If even a tiny labeled validation set exists (even 30–50 examples), compute AUC-PR and optimize threshold at the knee of the precision-recall curve.
  • Sensitivity analysis: Try contamination ∈ {0.01, 0.05, 0.10} and report flagged anomalies for human review; domain experts select the operationally acceptable false-positive rate.
  • Statistical heuristics: 3σ rule for Gaussian-ish data; beyond 1.5×IQR for boxplot-style detection.
The absence of labels makes this a calibration problem more than a model selection problem.
5.3 — Evaluating Unsupervised Models
Classification Medium
Match each evaluation scenario to the appropriate metric:

(a) You have ground-truth cluster labels and want to measure how well K-Means assignments agree
(b) No labels available; you want to measure cluster compactness and separation
(c) You used dimensionality reduction (PCA) before clustering and want to measure information preserved
(d) You trained an autoencoder for anomaly detection and want to quantify reconstruction fidelity
Answers
(a) Adjusted Rand Index (ARI) — measures agreement between predicted and true labels, adjusted for chance. Range [−1, 1]; 1 = perfect. Also: Normalized Mutual Information (NMI).
(b) Silhouette Score or Davies-Bouldin Index — Silhouette: high = compact well-separated clusters. Davies-Bouldin: lower = better (measures ratio of within-cluster scatter to between-cluster distance).
(c) Explained Variance Ratio — sum of explained variance of retained PCA components. Typically target ≥ 95% for downstream clustering.
(d) Reconstruction MSE / MAE on a held-out normal sample — low reconstruction error on normal data; high on anomalies. ROC-AUC if any labeled anomalies exist.

6.1 — Collaborative Filtering

Memory-Based vs Model-Based CF

Memory-based (User-User / Item-Item): compute cosine or Pearson similarity between users/items directly from the rating matrix. Simple but doesn't scale to millions of users/items. Model-based (Matrix Factorization): decompose the rating matrix R ≈ UVᵀ where U ∈ ℝ^(users×k), V ∈ ℝ^(items×k). Learn latent factors via ALS (alternating least squares) or SGD. Scales well and handles sparsity better.

MCQ Hard
A new user joins a streaming platform with no viewing history. Collaborative filtering recommends nothing useful. This is called the cold start problem. Which strategy is most directly targeted at solving it?
AIncrease the number of latent factors in matrix factorization to improve precision.
BUse ALS instead of SGD for more stable convergence of existing user embeddings.
CCollect minimal demographic or preference data at onboarding (hybrid approach), or fall back to content-based filtering using item features only.
DApply regularization to prevent overfitting to sparse user histories.
Explanation
Answer: C. Collaborative filtering requires historical interactions to compute similarities or learn embeddings — a brand-new user has no data for this. The fix is to use content-based filtering (recommends items similar to item attributes, not interaction history) until sufficient history accumulates, and/or to collect a brief onboarding quiz to initialize the user embedding. This hybrid approach — CF + CBF fallback — is the industry standard. Increasing latent factors (A) or using ALS (B) does nothing for users with no interactions.
Coding Hard
Implement user-user collaborative filtering from scratch. Given a user-item rating matrix (with NaN for unrated), compute cosine similarity between users and predict a target user's unrated items using a top-K neighborhood.
Reference Implementation
Python · NumPy
import numpy as np

def cosine_similarity_masked(R: np.ndarray) -> np.ndarray:
    """Compute pairwise cosine similarity ignoring NaN entries."""
    n_users = R.shape[0]
    sim = np.zeros((n_users, n_users))
    for u in range(n_users):
        for v in range(n_users):
            if u == v:
                sim[u, v] = 1.0; continue
            # Only items rated by BOTH users
            mask = ~np.isnan(R[u]) & ~np.isnan(R[v])
            if mask.sum() == 0: continue
            ru, rv = R[u, mask], R[v, mask]
            denom = np.linalg.norm(ru) * np.linalg.norm(rv)
            if denom > 0: sim[u, v] = (ru @ rv) / denom
    return sim


def predict_rating(R, sim, user_id, item_id, K=3) -> float:
    """Predict target user's rating for a given item."""
    # Get neighbors who rated this item
    rated_mask = ~np.isnan(R[:, item_id])
    rated_mask[user_id] = False  # exclude self

    neighbors  = np.where(rated_mask)[0]
    if len(neighbors) == 0: return np.nanmean(R[user_id])

    # Top-K most similar neighbors
    sims = sim[user_id, neighbors]
    topk = neighbors[np.argsort(sims)[::-1][:K]]
    topk_sims = sim[user_id, topk]

    numer = topk_sims @ R[topk, item_id]
    denom = np.abs(topk_sims).sum()
    return numer / denom if denom > 0 else np.nanmean(R[user_id])


# ── Demo ────────────────────────────────────────────
R = np.array([
    [5, 4, np.nan, 1],
    [4, np.nan, 4, 1],
    [np.nan, 5, 4, 2],
    [2, 1, np.nan, 5]
])

sim  = cosine_similarity_masked(R)
pred = predict_rating(R, sim, user_id=0, item_id=2, K=2)
print(f"Predicted rating for user 0, item 2: {pred:.2f}")

MCQ Medium
Content-based filtering recommends items similar to items a user has liked before, using item features. Which of the following is a key advantage of content-based over collaborative filtering?

6.3 — Recommender System Evaluation Metrics

Key Metrics Reference

RecSys evaluation has two regimes: rating prediction (use RMSE/MAE on held-out ratings) and ranking quality (use precision@K, recall@K, NDCG@K, MRR). The latter is more business-relevant since we care about top-K lists, not all ratings.

Precision@K = |relevant ∩ top-K| / K Recall@K = |relevant ∩ top-K| / |relevant| F1@K = 2 × P@K × R@K / (P@K + R@K) DCG@K = Σ_{i=1}^{K} rel_i / log₂(i+1) NDCG@K = DCG@K / IDCG@K (IDCG = ideal DCG, perfect ordering) MRR = (1/|queries|) Σ 1/rank_first_relevant
MCQ Hard
Two recommendation lists for the same user (3 relevant items total):

List A: [✓, ✗, ✓, ✗, ✓] — relevant items at positions 1, 3, 5
List B: [✓, ✓, ✓, ✗, ✗] — relevant items at positions 1, 2, 3

Which list has higher NDCG@5 and why?
AList A — it has the first item correct, which is highest weighted.
BList B — it concentrates all relevant items at the top, matching the ideal ordering and yielding DCG closest to IDCG.
CBoth are equal — both lists have the same number of relevant items in the top-5.
DList A — spreading recommendations avoids filter bubble, which NDCG penalizes.
Calculation
Answer: B. DCG discounts by log₂(rank+1), so higher ranks are worth more.

List A DCG = 1/log₂(2) + 1/log₂(4) + 1/log₂(6) ≈ 1.000 + 0.500 + 0.387 = 1.887
List B DCG = 1/log₂(2) + 1/log₂(3) + 1/log₂(4) ≈ 1.000 + 0.631 + 0.500 = 2.131
IDCG = List B DCG = 2.131 (this IS the ideal order)

NDCG_A = 1.887 / 2.131 ≈ 0.885 | NDCG_B = 2.131 / 2.131 = 1.000. List B wins.
Subjective Hard
You achieve RMSE = 0.85 on a test split of a movie rating dataset using matrix factorization. A colleague argues this is a good model. You argue RMSE alone is insufficient for a production recommender. Make your case.
Model Answer
RMSE measures rating prediction accuracy — how close predicted scores are to actual ratings. But in production, users don't see raw scores; they see ranked lists. A model with perfect RMSE could still produce terrible ranked recommendations if it consistently underestimates outlier loved movies that should appear at the top.

Missing dimensions from RMSE alone:
  • Ranking quality: NDCG@10 / Precision@10 measure whether top-displayed items are actually relevant.
  • Coverage: What fraction of the catalog gets recommended? A model exploiting only popular items has high coverage of clicks but poor diversity.
  • Novelty & Serendipity: Does the model surface non-obvious items the user would genuinely enjoy? Important for long-tail monetization.
  • Business metrics: CTR (click-through rate), dwell time, watch completion, return visit rate — the true north stars.
  • Fairness: Are niche genres / minority user groups receiving proportional recommendation quality?
A 0.85 RMSE from a naive popularity baseline would score poorly on all of the above. Offline metrics should always be validated against online A/B tests.
