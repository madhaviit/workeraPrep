## Chapter 0: Foundations You Must Know First

Before we can climb the mountain, we need to understand the ground beneath our feet.

What is a Neural Network?

A neural network is a mathematical function that maps inputs to outputs by composing many simpler functions called neurons. Each neuron computes a weighted sum of its inputs and passes the result through a non-linear function called an activation function.

Mathematically, a single neuron computes:

output = activation( w₁x₁ + w₂x₂ + ... + wₙxₙ + b )

Where: - x₁, x₂, ..., xₙ are input features - w₁, w₂, ..., wₙ are learnable weights - b is a learnable bias - activation(·) is a non-linear function like ReLU, sigmoid, or tanh

Why Non-Linearity?

Without non-linearity, stacking multiple layers reduces to a single linear transformation (matrix multiplication collapses into one matrix), making the network no more expressive than logistic regression. Non-linearities allow networks to approximate arbitrarily complex functions — this is the Universal Approximation Theorem.

Backpropagation in Plain English

Training means adjusting weights w to minimize a loss function — a measure of how wrong our predictions are. We use gradient descent: compute the derivative of the loss with respect to every weight, then nudge weights in the direction that reduces loss.

w ← w - η · ∂L/∂w

Where η (eta) is the learning rate — a small number like 0.001 that controls step size.

Backpropagation is the algorithm that efficiently computes all these derivatives using the chain rule of calculus, propagating error gradients backward through the network from output to input.

Tensors

All data in deep learning lives in tensors — multi-dimensional arrays. A scalar is a 0-D tensor, a vector is 1-D, a matrix is 2-D, and so on. PyTorch and TensorFlow both operate on tensors and automatically track gradients through them.

import torch

x = torch.tensor([[1.0, 2.0], [3.0, 4.0]], requires_grad=True)
y = (x ** 2).sum()
y.backward()
print(x.grad)  # dy/dx = 2x → [[2., 4.], [6., 8.]]

With that foundation, let us begin.

## Chapter 1: Transformer Models — The Architecture That Changed Everything

1.1 The Problem Transformers Solved

Before 2017, sequence modeling was dominated by RNNs and LSTMs. These architectures process sequences token by token, left to right. This creates two crushing limitations:

  1. Sequential bottleneck: You cannot process token 10 until you have processed tokens 1–9. This kills parallelism during training.
  2. Vanishing long-range dependencies: Information from token 1 must travel through hundreds of hidden states to influence token 500. Gradients vanish; the network forgets.

The paper “Attention Is All You Need” (Vaswani et al., 2017) proposed a radical idea: throw away recurrence entirely. Replace it with a mechanism called self-attention that allows every token to directly attend to every other token in the sequence simultaneously.

This was not an incremental improvement — it was a paradigm shift. Every modern large language model (GPT, BERT, T5, Claude, Gemini) is built on this architecture.

1.2 The Attention Mechanism — Ground Up

1.2.1 The Intuition: Queries, Keys, and Values

Imagine you are at a library. You have a query (what you’re looking for), the library has shelves of keys (catalog labels), and each shelf contains values (actual books). You match your query against every key, find the best matches, and retrieve the corresponding values — possibly a weighted blend of multiple books.

Attention does exactly this with vectors:

  • Query (Q): What am I looking for?
  • Key (K): What does each token advertise about itself?
  • Value (V): What information does each token actually carry?

For a sequence of n tokens, each token produces a query vector, a key vector, and a value vector. The output for each token is a weighted sum of all value vectors, where the weights are determined by how well that token’s query matches each other token’s key.

1.2.2 Scaled Dot-Product Attention — The Math

Given matrices Q (queries), K (keys), and V (values), attention is computed as:

Attention(Q, K, V) = softmax( Q·Kᵀ / √dₖ ) · V

Let us break this down step by step.

Step 1 — Compute similarity scores:

scores = Q · Kᵀ     # shape: [n_tokens × n_tokens]

scores[i][j] measures how much token i should attend to token j. This is just a dot product between the query of token i and the key of token j.

Step 2 — Scale by √dₖ:

scores = scores / √dₖ

Where dₖ is the dimensionality of the key vectors. Why scale? When dₖ is large, dot products grow large in magnitude, pushing softmax into regions with very small gradients (saturation). Dividing by √dₖ stabilizes training.

Step 3 — Apply softmax to get attention weights:

weights = softmax(scores, dim=-1)   # shape: [n_tokens × n_tokens]

Each row of weights sums to 1. weights[i][j] is the fraction of attention token i pays to token j.

Step 4 — Weighted sum of values:

output = weights · V   # shape: [n_tokens × d_v]

Each output token is a weighted blend of all value vectors.

1.2.3 Implementing Scaled Dot-Product Attention in PyTorch

import torch
import torch.nn as nn
import torch.nn.functional as F
import math

def scaled_dot_product_attention(Q, K, V, mask=None):
    """
    Args:
        Q: Queries,  shape [batch, heads, seq_len, d_k]
        K: Keys,     shape [batch, heads, seq_len, d_k]
        V: Values,   shape [batch, heads, seq_len, d_v]
        mask: Optional boolean mask, shape [batch, 1, seq_len, seq_len]
              True = position is masked (ignored)
    Returns:
        output:  shape [batch, heads, seq_len, d_v]
        weights: attention weights [batch, heads, seq_len, seq_len]
    """
    d_k = Q.size(-1)

    # Step 1 & 2: Scaled similarity scores
    # Q·Kᵀ → [batch, heads, seq_len_q, seq_len_k]
    scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(d_k)

    # Step 3: Apply mask (used in decoder to prevent attending to future tokens)
    if mask is not None:
        scores = scores.masked_fill(mask, float('-inf'))
        # -inf → 0 after softmax, effectively ignoring masked positions

    # Step 4: Softmax over the key dimension
    weights = F.softmax(scores, dim=-1)

    # Step 5: Weighted sum of values
    output = torch.matmul(weights, V)

    return output, weights

K = torch.randn(batch, heads, seq_len, d_k)
V = torch.randn(batch, heads, seq_len, d_k)

out, w = scaled_dot_product_attention(Q, K, V)
print(f"Output shape: {out.shape}")    # [2, 4, 10, 64]
print(f"Weights shape: {w.shape}")     # [2, 4, 10, 10]
print(f"Weights sum (should be 1): {w[0,0,0].sum():.4f}")

1.3 Multi-Head Attention

A single attention operation computes one type of relationship between tokens. But language is rich — “bank” relates to “river” in one context and “money” in another. We want the model to capture multiple types of relationships simultaneously.

Multi-Head Attention runs h independent attention operations (called heads) in parallel, each with its own learned Q, K, V projection matrices, then concatenates and re-projects the results.

MultiHead(Q, K, V) = Concat(head₁, head₂, ..., headₕ) · Wᴼ

where headᵢ = Attention(Q·Wᵢᴼ, K·Wᵢᴷ, V·Wᵢᵛ)

The key insight: instead of using d_model-dimensional keys, we project to d_model / h per head. This keeps the total computation roughly constant while multiplying the representational capacity.

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, num_heads):
        """
        d_model:    Total model dimension (e.g., 512)
        num_heads:  Number of parallel attention heads (e.g., 8)
        d_k = d_v = d_model / num_heads = 64 in this example
        """
        super().__init__()
        assert d_model % num_heads == 0, "d_model must be divisible by num_heads"

        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads   # dimension per head

        # Single projection matrices for all heads (efficient)
        # Instead of h separate W matrices, we use one large W and split
        self.W_q = nn.Linear(d_model, d_model)   # projects to [batch, seq, d_model]
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)   # output projection

    def split_heads(self, x):
        """
        Reshape from [batch, seq_len, d_model]
        to           [batch, num_heads, seq_len, d_k]
        """
        batch_size, seq_len, _ = x.size()
        # Reshape: [batch, seq, d_model] → [batch, seq, heads, d_k]
        x = x.view(batch_size, seq_len, self.num_heads, self.d_k)
        # Transpose: [batch, seq, heads, d_k] → [batch, heads, seq, d_k]
        return x.transpose(1, 2)

    def forward(self, query, key, value, mask=None):
        """
        query, key, value: [batch, seq_len, d_model]
        mask: optional [batch, 1, seq_len, seq_len]
        """
        batch_size = query.size(0)

        # 1. Linear projections
        Q = self.W_q(query)  # [batch, seq, d_model]
        K = self.W_k(key)
        V = self.W_v(value)

        # 2. Split into multiple heads
        Q = self.split_heads(Q)  # [batch, heads, seq, d_k]
        K = self.split_heads(K)
        V = self.split_heads(V)

        # 3. Scaled dot-product attention on each head in parallel
        attn_output, attn_weights = scaled_dot_product_attention(Q, K, V, mask)
        # attn_output: [batch, heads, seq, d_k]

        # 4. Concatenate heads
        # Transpose back: [batch, heads, seq, d_k] → [batch, seq, heads, d_k]
        attn_output = attn_output.transpose(1, 2).contiguous()
        # Merge heads: [batch, seq, heads, d_k] → [batch, seq, d_model]
        attn_output = attn_output.view(batch_size, -1, self.d_model)

        # 5. Final output projection
        output = self.W_o(attn_output)  # [batch, seq, d_model]

        return output, attn_weights

print(f"Attn weights: {weights.shape}")  # [2, 8, 20, 20]

1.4 Positional Encoding

Attention has no built-in notion of order — it treats the input as a set of tokens, not a sequence. We must inject positional information explicitly.

The original Transformer used fixed sinusoidal encodings:

PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

Where pos is the position index and i is the dimension index. Different dimensions encode different frequency sine/cosine waves, allowing the model to distinguish positions and even extrapolate to unseen lengths.

class PositionalEncoding(nn.Module):
    def __init__(self, d_model, max_seq_len=5000, dropout=0.1):
        super().__init__()
        self.dropout = nn.Dropout(dropout)

        # Create a matrix of shape [max_seq_len, d_model]
        pe = torch.zeros(max_seq_len, d_model)

        # Position indices: [0, 1, 2, ..., max_seq_len-1]
        position = torch.arange(0, max_seq_len).unsqueeze(1).float()

        # Division term: 10000^(2i/d_model), computed in log space for stability
        div_term = torch.exp(
            torch.arange(0, d_model, 2).float() * (-math.log(10000.0) / d_model)
        )

        # Even dimensions: sine, odd dimensions: cosine
        pe[:, 0::2] = torch.sin(position * div_term)   # 0, 2, 4, ...
        pe[:, 1::2] = torch.cos(position * div_term)   # 1, 3, 5, ...

        # Add batch dimension and register as non-trainable buffer
        pe = pe.unsqueeze(0)  # [1, max_seq_len, d_model]
        self.register_buffer('pe', pe)

    def forward(self, x):
        """
        x: [batch, seq_len, d_model]
        Add positional encoding to token embeddings
        """
        x = x + self.pe[:, :x.size(1), :]
        return self.dropout(x)

1.5 The Feed-Forward Sublayer

After attention, each position is independently processed by a small 2-layer MLP:

FFN(x) = max(0, x·W₁ + b₁)·W₂ + b₂

The inner dimension is typically 4× the model dimension (e.g., 2048 for d_model=512). This is where most of the “knowledge storage” happens — research has shown that these layers act as key-value memories.

class FeedForward(nn.Module):
    def __init__(self, d_model, d_ff, dropout=0.1):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(d_model, d_ff),
            nn.ReLU(),
            nn.Dropout(dropout),
            nn.Linear(d_ff, d_model),
            nn.Dropout(dropout),
        )

    def forward(self, x):
        return self.net(x)

1.6 Layer Normalization and Residual Connections

Each sublayer (attention and FFN) is wrapped in a residual connection and layer normalization:

output = LayerNorm(x + Sublayer(x))

Why residuals? They create gradient highways — during backprop, gradients can flow directly from output to input without passing through every layer, preventing vanishing gradients in deep networks.

Why layer norm? It normalizes the activations across the feature dimension for each token independently, stabilizing training without the batch-size dependency of batch normalization.

1.7 The Full Transformer Encoder Layer

class TransformerEncoderLayer(nn.Module):
    def __init__(self, d_model, num_heads, d_ff, dropout=0.1):
        super().__init__()
        self.self_attn = MultiHeadAttention(d_model, num_heads)
        self.ffn = FeedForward(d_model, d_ff, dropout)
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        self.dropout = nn.Dropout(dropout)

    def forward(self, x, mask=None):
        # --- Sub-layer 1: Self-Attention ---
        attn_out, _ = self.self_attn(x, x, x, mask)   # Q=K=V=x
        x = self.norm1(x + self.dropout(attn_out))     # Residual + Norm

        # --- Sub-layer 2: Feed-Forward ---
        ffn_out = self.ffn(x)
        x = self.norm2(x + ffn_out)                    # Residual + Norm

        return x

1.8 The Encoder-Decoder Architecture

The original Transformer was designed for sequence-to-sequence tasks (e.g., translation). It has two components:

1.8.1 The Encoder

The encoder reads the input sequence and produces rich contextual representations. It stacks N encoder layers (typically N=6). Each encoder layer applies self-attention (where every input token can attend to every other input token) and a feed-forward network.

Input → Embedding + Positional Encoding
      → [EncoderLayer₁ → EncoderLayer₂ → ... → EncoderLayerₙ]
      → Encoder Output (contextual representations)

1.8.2 The Decoder

The decoder generates the output sequence token by token. Each decoder layer has three sublayers:

  1. Masked Self-Attention: The decoder attends to its own previously generated tokens. The “masking” prevents attending to future positions (causal masking), since at inference time, future tokens don’t exist yet.

  2. Cross-Attention: This is the encoder-decoder bridge. The decoder’s queries attend to the encoder’s keys and values. This is how the decoder “reads” the input — the decoder asks questions (queries) and the encoder provides answers (keys + values).

  3. Feed-Forward Network: Same as encoder.

class TransformerDecoderLayer(nn.Module):
    def __init__(self, d_model, num_heads, d_ff, dropout=0.1):
        super().__init__()
        # Sub-layer 1: Masked self-attention (causal)
        self.self_attn = MultiHeadAttention(d_model, num_heads)
        # Sub-layer 2: Cross-attention over encoder output
        self.cross_attn = MultiHeadAttention(d_model, num_heads)
        # Sub-layer 3: Feed-forward
        self.ffn = FeedForward(d_model, d_ff, dropout)

        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        self.norm3 = nn.LayerNorm(d_model)
        self.dropout = nn.Dropout(dropout)

    def forward(self, x, encoder_output, src_mask=None, tgt_mask=None):
        """
        x:              Decoder input, [batch, tgt_len, d_model]
        encoder_output: [batch, src_len, d_model]
        src_mask:       Padding mask for encoder output
        tgt_mask:       Causal mask for decoder input (upper-triangular)
        """
        # --- Sub-layer 1: Masked self-attention ---
        # Q, K, V all come from decoder input; mask prevents future-peeking
        attn1, _ = self.self_attn(x, x, x, tgt_mask)
        x = self.norm1(x + self.dropout(attn1))

        # --- Sub-layer 2: Cross-attention ---
        # Queries from decoder, Keys & Values from encoder output
        attn2, cross_weights = self.cross_attn(
            query=x,
            key=encoder_output,
            value=encoder_output,
            mask=src_mask
        )
        x = self.norm2(x + self.dropout(attn2))

        # --- Sub-layer 3: Feed-forward ---
        ffn_out = self.ffn(x)
        x = self.norm3(x + ffn_out)

        return x, cross_weights


def make_causal_mask(seq_len, device):
    """
    Creates an upper-triangular mask to prevent attending to future positions.
    mask[i][j] = True means token i should NOT attend to token j.
    """
    mask = torch.triu(torch.ones(seq_len, seq_len, device=device), diagonal=1).bool()
    return mask.unsqueeze(0).unsqueeze(0)  # [1, 1, seq, seq]

1.8.3 The Complete Encoder-Decoder Model

class Transformer(nn.Module):
    def __init__(self, src_vocab, tgt_vocab, d_model=512, num_heads=8,
                 num_layers=6, d_ff=2048, max_seq_len=512, dropout=0.1):
        super().__init__()

        # Embeddings (shared or separate for src/tgt)
        self.src_embedding = nn.Embedding(src_vocab, d_model)
        self.tgt_embedding = nn.Embedding(tgt_vocab, d_model)
        self.pos_encoding = PositionalEncoding(d_model, max_seq_len, dropout)

        # Encoder stack
        self.encoder_layers = nn.ModuleList([
            TransformerEncoderLayer(d_model, num_heads, d_ff, dropout)
            for _ in range(num_layers)
        ])

        # Decoder stack
        self.decoder_layers = nn.ModuleList([
            TransformerDecoderLayer(d_model, num_heads, d_ff, dropout)
            for _ in range(num_layers)
        ])

        # Final projection to vocabulary
        self.fc_out = nn.Linear(d_model, tgt_vocab)
        self.d_model = d_model

    def encode(self, src, src_mask=None):
        x = self.pos_encoding(self.src_embedding(src) * math.sqrt(self.d_model))
        for layer in self.encoder_layers:
            x = layer(x, src_mask)
        return x  # [batch, src_len, d_model]

    def decode(self, tgt, encoder_output, src_mask=None, tgt_mask=None):
        x = self.pos_encoding(self.tgt_embedding(tgt) * math.sqrt(self.d_model))
        for layer in self.decoder_layers:
            x, _ = layer(x, encoder_output, src_mask, tgt_mask)
        return x  # [batch, tgt_len, d_model]

    def forward(self, src, tgt, src_mask=None):
        tgt_mask = make_causal_mask(tgt.size(1), tgt.device)
        encoder_output = self.encode(src, src_mask)
        decoder_output = self.decode(tgt, encoder_output, src_mask, tgt_mask)
        logits = self.fc_out(decoder_output)
        return logits  # [batch, tgt_len, tgt_vocab]

1.9 Variants: Encoder-Only, Decoder-Only

  • Encoder-only (BERT-style): No decoder. Bidirectional attention — each token sees all others. Used for understanding tasks: classification, NER, question answering. Pretrained with Masked Language Modeling.

  • Decoder-only (GPT-style): No encoder. Causal attention only. Used for generation. Pretrained with next-token prediction. This is the architecture behind ChatGPT, Claude, and LLaMA.

  • Encoder-decoder (T5, BART): Full architecture as above. Used for translation, summarization, any seq2seq task.

1.10 Why Attention Works: The Mechanics

The power of attention can be understood from three angles:

  1. Arbitrary range: Any two tokens can directly influence each other in a single layer. Path length between any pair is O(1), versus O(n) for RNNs.

  2. Dynamic computation: Attention weights are input-dependent. The same word computes different relationships in different contexts. “Bank” in “river bank” has different keys/queries than “bank” in “central bank.”

  3. Interpretability: Attention weights provide a window into what the model “looks at,” making Transformers more interpretable than RNNs (though this is a simplification — the relationship between attention and model behavior is complex).

## Chapter 2: Graph Neural Networks — Learning on Structured Data

2.1 Why Graphs?

Most real-world data is not a sequence or a grid — it is a graph:

  • Social networks (users = nodes, friendships = edges)
  • Molecules (atoms = nodes, bonds = edges)
  • Knowledge graphs (entities = nodes, relations = edges)
  • Road networks (intersections = nodes, roads = edges)
  • Citation networks (papers = nodes, citations = edges)
  • Protein interaction networks

CNNs exploit the grid structure of images. RNNs exploit the sequential structure of language. GNNs are the equivalent framework for arbitrary graph structures.

2.2 Graph Basics

A graph G = (V, E) consists of: - V: a set of n nodes (also called vertices) - E: a set of edges between pairs of nodes - X ∈ ℝⁿˣᶠ: a feature matrix where row i is the feature vector of node i - A ∈ ℝⁿˣⁿ: an adjacency matrix where A[i][j] = 1 if there’s an edge from i to j

A graph can be directed (edges have direction) or undirected (edges are symmetric). Edges can also have features.

2.3 The Core Idea: Message Passing

The central operation in virtually all GNNs is message passing. The intuition: a node should update its representation based on information gathered from its neighbors. This is analogous to asking: “Given who I am and who I am connected to, how should I update my understanding of myself?”

The message-passing framework is:

Step 1 — Message: Each neighbor sends a message to the central node.

mᵢⱼ = MESSAGE(hᵢᵗ, hⱼᵗ, eᵢⱼ)

Step 2 — Aggregate: Collect messages from all neighbors into a single summary.

Mᵢᵗ = AGGREGATE({mᵢⱼ : j ∈ N(i)})

Step 3 — Update: Update the node’s own representation.

hᵢᵗ⁺¹ = UPDATE(hᵢᵗ, Mᵢᵗ)

After k rounds of message passing, each node’s representation encodes information from its k-hop neighborhood.

2.4 Graph Convolutional Network (GCN)

The seminal GCN paper (Kipf & Welling, 2017) introduced a particularly elegant form of message passing.

The Spectral Motivation: Graph convolutions can be derived from spectral graph theory as a localized approximation of graph signal filtering. But the practical result is beautifully simple.

For a single GCN layer:

H^(l+1) = σ( D̃^(-1/2) Ã D̃^(-1/2) H^(l) W^(l) )

Where: - Ã = A + I (adjacency matrix with added self-loops) - D̃[i][i] = Σⱼ Ã[i][j] (degree matrix of Ã) - D̃^(-1/2) Ã D̃^(-1/2) is the symmetric normalized adjacency - W^(l) is the learnable weight matrix for layer l - σ is an activation function (e.g., ReLU)

The D̃^(-1/2) Ã D̃^(-1/2) term normalizes by node degree, preventing high-degree nodes from drowning out low-degree nodes. The Ã = A + I self-loops ensure each node also retains its own features.

import torch
import torch.nn as nn
import torch.nn.functional as F

class GCNLayer(nn.Module):
    def __init__(self, in_features, out_features):
        super().__init__()
        self.linear = nn.Linear(in_features, out_features, bias=False)
        # Initialize weights following Glorot uniform
        nn.init.xavier_uniform_(self.linear.weight)

    def forward(self, H, A_hat):
        """
        H:     Node feature matrix [n_nodes, in_features]
        A_hat: Pre-normalized adjacency D̃^(-1/2) Ã D̃^(-1/2) [n_nodes, n_nodes]
        """
        # Aggregate neighbor features: A_hat @ H
        aggregated = torch.mm(A_hat, H)           # [n_nodes, in_features]
        # Linear transformation + activation
        return self.linear(aggregated)            # [n_nodes, out_features]


def normalize_adjacency(A):
    """
    Compute D̃^(-1/2) Ã D̃^(-1/2) for GCN.
    A: adjacency matrix [n, n] (can be torch.Tensor or scipy sparse)
    """
    n = A.size(0)
    # Add self-loops: Ã = A + I
    A_tilde = A + torch.eye(n, device=A.device)
    # Compute degree matrix D̃
    D_tilde = A_tilde.sum(dim=1)  # [n] — row sums
    # D̃^(-1/2) as a vector (we'll apply it as element-wise scaling)
    D_inv_sqrt = D_tilde.pow(-0.5)
    D_inv_sqrt[D_inv_sqrt == float('inf')] = 0  # handle isolated nodes
    # Symmetric normalization: D̃^(-1/2) Ã D̃^(-1/2)
    # Equivalent to: scale rows and columns by D_inv_sqrt
    A_hat = D_inv_sqrt.unsqueeze(1) * A_tilde * D_inv_sqrt.unsqueeze(0)
    return A_hat


class GCN(nn.Module):
    def __init__(self, in_features, hidden_dim, num_classes, dropout=0.5):
        super().__init__()
        self.gcn1 = GCNLayer(in_features, hidden_dim)
        self.gcn2 = GCNLayer(hidden_dim, num_classes)
        self.dropout = nn.Dropout(dropout)

    def forward(self, X, A):
        A_hat = normalize_adjacency(A)
        # Layer 1: GCN + ReLU + Dropout
        h = F.relu(self.gcn1(X, A_hat))
        h = self.dropout(h)
        # Layer 2: GCN (no activation — logits for classification)
        logits = self.gcn2(h, A_hat)
        return logits  # [n_nodes, num_classes]

2.5 Graph Attention Networks (GAT)

GCN uses fixed structural weights (based on degree). But not all neighbors are equally important. GAT (Veličković et al., 2018) learns to weight neighbors dynamically — just like Transformer attention.

For each edge (i, j), GAT computes an attention coefficient:

eᵢⱼ = LeakyReLU( aᵀ [Whᵢ || Whⱼ] )
αᵢⱼ = softmax_j(eᵢⱼ) = exp(eᵢⱼ) / Σ_{k∈N(i)} exp(eᵢₖ)

Then aggregates:

hᵢ' = σ( Σ_{j∈N(i)} αᵢⱼ · W·hⱼ )
class GATLayer(nn.Module):
    def __init__(self, in_features, out_features, num_heads=1,
                 dropout=0.6, alpha=0.2, concat=True):
        """
        in_features:  Input feature dimension
        out_features: Output feature dimension per head
        num_heads:    Number of attention heads (multi-head GAT)
        alpha:        Negative slope for LeakyReLU
        concat:       If True, concatenate heads; if False, average them
        """
        super().__init__()
        self.num_heads = num_heads
        self.out_features = out_features
        self.concat = concat

        # One weight matrix per head
        self.W = nn.Parameter(torch.FloatTensor(num_heads, in_features, out_features))
        # Attention vector per head: [2 * out_features] → scalar
        self.a = nn.Parameter(torch.FloatTensor(num_heads, 2 * out_features))

        self.leaky_relu = nn.LeakyReLU(alpha)
        self.dropout = nn.Dropout(dropout)

        # Initialize
        nn.init.xavier_uniform_(self.W)
        nn.init.xavier_uniform_(self.a.unsqueeze(0))

    def forward(self, H, adj):
        """
        H:   [n_nodes, in_features]
        adj: [n_nodes, n_nodes] adjacency matrix (binary)
        """
        n = H.size(0)
        outputs = []

        for k in range(self.num_heads):
            # Linear transformation: [n, in_features] @ [in_features, out_features]
            Wh = H @ self.W[k]  # [n, out_features]

            # Compute attention scores for all pairs (i, j)
            # Expand for broadcasting: Wh_i for source, Wh_j for target
            Wh_repeat_i = Wh.unsqueeze(1).expand(n, n, -1)  # [n, n, out]
            Wh_repeat_j = Wh.unsqueeze(0).expand(n, n, -1)  # [n, n, out]

            # Concatenate along feature dim: [n, n, 2*out]
            concat_features = torch.cat([Wh_repeat_i, Wh_repeat_j], dim=-1)

            # Apply attention vector: [n, n, 2*out] @ [2*out] → [n, n]
            e = self.leaky_relu((concat_features * self.a[k]).sum(dim=-1))

            # Mask non-edges with -inf (so they get 0 after softmax)
            mask = (adj == 0)
            e = e.masked_fill(mask, float('-inf'))

            # Softmax over neighbors
            alpha = F.softmax(e, dim=1)    # [n, n]
            alpha = self.dropout(alpha)

            # Weighted aggregation
            h_prime = alpha @ Wh           # [n, out_features]
            outputs.append(h_prime)

        if self.concat:
            return F.elu(torch.cat(outputs, dim=-1))      # [n, heads*out]
        else:
            return F.elu(torch.stack(outputs, dim=0).mean(0))  # [n, out]

2.6 Message-Passing Neural Networks (MPNN) — The General Framework

PyTorch Geometric (PyG) provides an elegant base class for message passing. Let’s implement it from scratch to understand the mechanics:

# Using PyTorch Geometric's MessagePassing base class
from torch_geometric.nn import MessagePassing
from torch_geometric.utils import add_self_loops, degree

class CustomGCNConv(MessagePassing):
    """
    A from-scratch GCN convolution using PyG's MessagePassing framework.
    aggr='add' means we sum incoming messages (GCN uses sum, not mean)
    """
    def __init__(self, in_channels, out_channels):
        super().__init__(aggr='add')
        self.lin = nn.Linear(in_channels, out_channels, bias=False)

    def forward(self, x, edge_index):
        """
        x:          [n_nodes, in_channels]
        edge_index: [2, n_edges] — COO format edges (source, target)
        """
        # Step 1: Add self-loops
        edge_index, _ = add_self_loops(edge_index, num_nodes=x.size(0))

        # Step 2: Linear transform of node features
        x = self.lin(x)   # [n_nodes, out_channels]

        # Step 3: Compute degree-based normalization
        row, col = edge_index
        deg = degree(col, x.size(0), dtype=x.dtype)  # [n_nodes]
        deg_inv_sqrt = deg.pow(-0.5)
        norm = deg_inv_sqrt[row] * deg_inv_sqrt[col]  # per-edge norm factor

        # Step 4: Propagate messages
        # This calls message() → aggregate() → update() internally
        return self.propagate(edge_index, x=x, norm=norm)

    def message(self, x_j, norm):
        """
        Called for each edge. x_j is the source node feature.
        norm: per-edge normalization factor.
        Returns the message from source node j to target node i.
        """
        return norm.view(-1, 1) * x_j   # scale source features by norm

    def update(self, aggr_out):
        """
        Called after aggregation. aggr_out is the aggregated message.
        We just return it here; in more complex models, apply an MLP.
        """
        return aggr_out

2.7 GNNs on Dynamic and Temporal Graphs

Real graphs change over time: friendships form and dissolve, transactions occur at specific times, proteins interact dynamically. Static GNNs cannot capture this temporal dimension. Two main paradigms handle it:

2.7.1 Discrete-Time Dynamic Graphs (DTDG)

The graph is observed as a sequence of snapshots at discrete time steps: G₁, G₂, …, Gₜ. We apply a GNN to each snapshot and use an RNN/LSTM to model temporal evolution.

import torch
import torch.nn as nn

class TemporalGCN(nn.Module):
    """
    For each time step, applies a GCN, then uses GRU to propagate temporal info.
    Architecture: GCN(t) → GRU → node embeddings at time t
    """
    def __init__(self, node_features, gcn_out, gru_hidden, num_classes):
        super().__init__()
        # Shared GCN applied at each time step
        self.gcn = GCN(node_features, gcn_out, gcn_out)
        # GRU takes GCN output at each step and updates temporal state
        self.gru = nn.GRU(gcn_out, gru_hidden, batch_first=True)
        # Classification head
        self.classifier = nn.Linear(gru_hidden, num_classes)

    def forward(self, X_sequence, A_sequence):
        """
        X_sequence: List of T feature matrices, each [n_nodes, node_features]
        A_sequence: List of T adjacency matrices, each [n_nodes, n_nodes]
        Returns:    Node predictions at the final time step
        """
        T = len(X_sequence)
        n_nodes = X_sequence[0].size(0)

        # Apply GCN at each snapshot → collect temporal node embeddings
        gcn_outputs = []
        for t in range(T):
            h_t = self.gcn(X_sequence[t], A_sequence[t])   # [n_nodes, gcn_out]
            gcn_outputs.append(h_t)

        # Stack into sequence: [n_nodes, T, gcn_out]
        # Each node has a sequence of GCN embeddings through time
        node_sequences = torch.stack(gcn_outputs, dim=1)

        # GRU across time for each node
        gru_out, _ = self.gru(node_sequences)     # [n_nodes, T, gru_hidden]
        final_embeddings = gru_out[:, -1, :]      # [n_nodes, gru_hidden]

        return self.classifier(final_embeddings)  # [n_nodes, num_classes]

2.7.2 Continuous-Time Dynamic Graphs (CTDG)

Interactions happen at arbitrary continuous timestamps, not discrete snapshots. The graph evolves as a stream of (source, target, time, features) events. This calls for more sophisticated methods like:

Temporal Point Processes: Model the time between events as a stochastic process.

TGNN (Temporal Graph Neural Network): Maintain per-node memory states that are updated upon each interaction. The seminal model here is TGN (Temporal Graph Networks, Rossi et al., 2020).

class TGNMemory(nn.Module):
    """
    Simplified TGN memory module.
    Each node has a memory vector that gets updated when it interacts.
    """
    def __init__(self, num_nodes, memory_dim, message_dim, time_dim=1):
        super().__init__()
        self.memory_dim = memory_dim
        # Per-node memory: initialized to zeros
        self.register_buffer('memory', torch.zeros(num_nodes, memory_dim))
        # Time encoder: maps scalar time delta to a vector
        self.time_encoder = nn.Linear(1, time_dim)
        # Message function: [memory_src || memory_dst || time_enc || edge_feat] → message
        self.message_fn = nn.Linear(2 * memory_dim + time_dim + message_dim, memory_dim)
        # GRU cell to update memory
        self.memory_updater = nn.GRUCell(memory_dim, memory_dim)

    def get_memory(self, node_ids):
        return self.memory[node_ids]

    def update_memory(self, src_ids, dst_ids, timestamps, edge_features):
        """
        Update memory for source and destination nodes after an interaction.
        """
        src_mem = self.memory[src_ids]      # [batch, memory_dim]
        dst_mem = self.memory[dst_ids]
        time_enc = self.time_encoder(timestamps.unsqueeze(-1))  # [batch, time_dim]

        # Compute messages for src and dst
        msg_src = self.message_fn(
            torch.cat([src_mem, dst_mem, time_enc, edge_features], dim=-1)
        )
        msg_dst = self.message_fn(
            torch.cat([dst_mem, src_mem, time_enc, edge_features], dim=-1)
        )

        # Update memories with GRU
        self.memory[src_ids] = self.memory_updater(msg_src, src_mem)
        self.memory[dst_ids] = self.memory_updater(msg_dst, dst_mem)

## Chapter 3: Convolutional Neural Networks — Vision from First Principles

3.1 The Problem with Fully-Connected Networks for Images

Consider a 224×224 RGB image. As a flat vector, this has 224 × 224 × 3 = 150,528 dimensions. A single fully-connected layer mapping to 1000 hidden units would need 150,528 × 1,000 = 150 million parameters — and we’d have multiple such layers. This is computationally intractable and prone to catastrophic overfitting.

Worse, fully-connected layers are not equivariant to translation: if a cat moves 10 pixels to the right, the network sees a completely different input and must learn the same cat detector all over again.

CNNs solve both problems through: 1. Local connectivity: Each neuron only connects to a small spatial region (the receptive field) 2. Weight sharing: The same filter (set of weights) is applied at every position 3. Hierarchical features: Early layers detect edges, middle layers detect textures, deep layers detect objects

3.2 The Convolution Operation — From Scratch

3.2.1 What is a Convolution?

A convolution slides a small filter (kernel) across the input, computing a dot product at each position. The result is a feature map indicating where and how strongly that filter’s pattern is present.

For a 2D input I and kernel K of size f×f, the output at position (i, j) is:

Output[i][j] = Σₘ Σₙ I[i+m][j+n] × K[m][n]

In CNNs, we have multiple input channels (e.g., 3 for RGB) and multiple output filters (e.g., 64). For input X of shape [C_in, H, W] and kernel K of shape [C_out, C_in, f, f]:

Output[c_out, i, j] = Σ_{c_in} Σₘ Σₙ X[c_in, i+m, j+n] × K[c_out, c_in, m, n]
import torch
import torch.nn as nn
import torch.nn.functional as F
import numpy as np

def manual_conv2d(input, kernel, stride=1, padding=0):
    """
    Manual implementation of 2D convolution to understand internals.
    input:  [batch, C_in, H, W]
    kernel: [C_out, C_in, kH, kW]
    Returns: [batch, C_out, H_out, W_out]
    """
    batch, C_in, H, W = input.shape
    C_out, _, kH, kW = kernel.shape

    # Apply padding
    if padding > 0:
        input = F.pad(input, [padding]*4)  # pad all 4 sides

    # Output spatial dimensions
    H_out = (H + 2*padding - kH) // stride + 1
    W_out = (W + 2*padding - kW) // stride + 1

    output = torch.zeros(batch, C_out, H_out, W_out)

    for b in range(batch):
        for c_out in range(C_out):
            for i in range(H_out):
                for j in range(W_out):
                    h_start = i * stride
                    w_start = j * stride
                    # Extract patch
                    patch = input[b, :, h_start:h_start+kH, w_start:w_start+kW]
                    # Element-wise multiply and sum (dot product)
                    output[b, c_out, i, j] = (patch * kernel[c_out]).sum()

    return output

3.2.2 Padding

Without padding, convolutions shrink the spatial dimensions: an n×n input with an f×f kernel produces (n-f+1)×(n-f+1) output. Deep networks would rapidly collapse to size 1.

Valid padding (padding=0): No padding, output is smaller than input. Same padding (padding=(f-1)/2): Pad so output is same size as input.

With f=3, padding=1: H_out = H_in
With f=5, padding=2: H_out = H_in
General rule: padding = (f-1) / 2 for odd kernel sizes

3.2.3 Stride

Stride controls how many pixels the filter jumps at each step. stride=1 moves one pixel at a time (default). stride=2 moves two pixels, halving the output size.

H_out = floor((H_in + 2*padding - f) / stride) + 1

Stride > 1 is an alternative to pooling for downsampling — it’s differentiable and can be learned.

3.2.4 Dilation

Dilation inserts spaces between kernel elements, effectively increasing the receptive field without adding parameters and without losing spatial resolution.

A 3×3 kernel with dilation=2 has the same number of parameters but sees a 5×5 area (with gaps). Dilation=1 is standard convolution.

Effective kernel size = f + (f-1)(d-1) = d*(f-1) + 1
For f=3, d=2: effective size = 5
For f=3, d=4: effective size = 9

Dilated convolutions are widely used in semantic segmentation (DeepLab) and audio generation (WaveNet).

class ConvBlock(nn.Module):
    """
    A comprehensive conv block showcasing padding, stride, and dilation.
    """
    def __init__(self, in_channels, out_channels, kernel_size=3,
                 stride=1, padding=1, dilation=1):
        super().__init__()
        # When dilation > 1, adjust padding to keep output size stable:
        # effective_padding = dilation * (kernel_size - 1) / 2
        self.conv = nn.Conv2d(
            in_channels, out_channels,
            kernel_size=kernel_size,
            stride=stride,
            padding=padding,
            dilation=dilation,
            bias=False   # Usually no bias when using BatchNorm
        )
        self.bn = nn.BatchNorm2d(out_channels)
        self.relu = nn.ReLU(inplace=True)

    def forward(self, x):
        return self.relu(self.bn(self.conv(x)))

3.3 Pooling

Pooling reduces spatial dimensions, making representations more compact and somewhat translation-invariant.

Max Pooling: Takes the maximum value in each pooling window. Picks the most active feature. Most commonly used.

Average Pooling: Takes the mean. Smoother but less sharp.

Global Average Pooling (GAP): Averages across all spatial positions, producing a [batch, channels] tensor from [batch, channels, H, W]. Used at the end of modern CNNs instead of fully-connected layers — drastically reduces parameters and acts as regularization.

3.4 Implementing a Complete CNN for Image Classification

Let us build a modern, fully-functional CNN for image classification, following the design principles of ResNet.

3.4.1 The Vanishing Gradient Problem in Deep CNNs

Before ResNet (2015), training very deep networks (>20 layers) was notoriously difficult. Gradients exponentially shrink as they propagate back through many layers, making early layers receive near-zero gradients and thus barely learning.

3.4.2 Residual Connections — The Solution

He et al. (2015) introduced the residual block:

output = F(x, {Wᵢ}) + x

Instead of learning the desired mapping H(x), the network learns the residual F(x) = H(x) - x. The identity shortcut + x creates a gradient highway: during backprop, the gradient can flow directly through the identity connection, bypassing the conv layers.

class ResidualBlock(nn.Module):
    """
    Standard ResNet residual block.
    If in_channels != out_channels, uses a 1×1 conv as shortcut projection.
    """
    expansion = 1  # Basic block doesn't expand channels

    def __init__(self, in_channels, out_channels, stride=1):
        super().__init__()

        # Main path: 3×3 conv → BN → ReLU → 3×3 conv → BN
        self.conv1 = nn.Conv2d(in_channels, out_channels,
                               kernel_size=3, stride=stride, padding=1, bias=False)
        self.bn1 = nn.BatchNorm2d(out_channels)

        self.conv2 = nn.Conv2d(out_channels, out_channels,
                               kernel_size=3, stride=1, padding=1, bias=False)
        self.bn2 = nn.BatchNorm2d(out_channels)

        self.relu = nn.ReLU(inplace=True)

        # Shortcut: if dimensions change, project to match
        self.shortcut = nn.Sequential()
        if stride != 1 or in_channels != out_channels:
            self.shortcut = nn.Sequential(
                nn.Conv2d(in_channels, out_channels,
                          kernel_size=1, stride=stride, bias=False),
                nn.BatchNorm2d(out_channels)
            )

    def forward(self, x):
        # Main path
        residual = self.shortcut(x)    # Save for skip connection (may be projected)

        out = self.relu(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))  # No ReLU yet — add residual first

        out += residual                # Residual addition
        out = self.relu(out)           # ReLU after addition

        return out


class ResNet(nn.Module):
    """
    Simplified ResNet for CIFAR-10 (32×32 images, 10 classes).
    Adapted from ResNet architecture, scaled for small images.
    """
    def __init__(self, num_classes=10):
        super().__init__()

        # Initial feature extraction
        self.stem = nn.Sequential(
            nn.Conv2d(3, 64, kernel_size=3, stride=1, padding=1, bias=False),
            nn.BatchNorm2d(64),
            nn.ReLU(inplace=True)
        )
        # Note: No max pooling here (images are 32×32, not 224×224)

        # Residual stages
        self.stage1 = self._make_stage(64, 64, num_blocks=2, stride=1)
        self.stage2 = self._make_stage(64, 128, num_blocks=2, stride=2)
        self.stage3 = self._make_stage(128, 256, num_blocks=2, stride=2)
        self.stage4 = self._make_stage(256, 512, num_blocks=2, stride=2)

        # Global average pooling + classifier
        self.global_pool = nn.AdaptiveAvgPool2d((1, 1))
        self.classifier = nn.Linear(512, num_classes)

        # Weight initialization
        self._init_weights()

    def _make_stage(self, in_channels, out_channels, num_blocks, stride):
        """Create a stage: first block may downsample, rest maintain size."""
        blocks = [ResidualBlock(in_channels, out_channels, stride=stride)]
        for _ in range(1, num_blocks):
            blocks.append(ResidualBlock(out_channels, out_channels, stride=1))
        return nn.Sequential(*blocks)

    def _init_weights(self):
        for m in self.modules():
            if isinstance(m, nn.Conv2d):
                # He initialization: accounts for ReLU non-linearity
                nn.init.kaiming_normal_(m.weight, mode='fan_out', nonlinearity='relu')
            elif isinstance(m, nn.BatchNorm2d):
                nn.init.constant_(m.weight, 1)
                nn.init.constant_(m.bias, 0)

    def forward(self, x):
        # x: [batch, 3, 32, 32]
        x = self.stem(x)           # [batch, 64, 32, 32]
        x = self.stage1(x)         # [batch, 64, 32, 32]
        x = self.stage2(x)         # [batch, 128, 16, 16]

        x = self.global_pool(x)    # [batch, 512, 1, 1]
        x = x.flatten(1)           # [batch, 512]
        return self.classifier(x)  # [batch, num_classes]

    model = model.to(device)

    # SGD with momentum and weight decay — the classic combo for CNNs
    optimizer = torch.optim.SGD(
        model.parameters(), lr=lr,
        momentum=0.9, weight_decay=1e-4, nesterov=True
    )
    # Cosine annealing: smoothly decay LR to 0
    scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=num_epochs)
    criterion = nn.CrossEntropyLoss(label_smoothing=0.1)  # Label smoothing regularization

    for epoch in range(num_epochs):
        model.train()
        total_loss, correct, total = 0, 0, 0

        for images, labels in train_loader:
            images, labels = images.to(device), labels.to(device)
            optimizer.zero_grad()
            logits = model(images)
            loss = criterion(logits, labels)
            loss.backward()
            # Gradient clipping: prevents exploding gradients
            torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
            optimizer.step()

            total_loss += loss.item()
            correct += (logits.argmax(1) == labels).sum().item()
            total += labels.size(0)

        scheduler.step()
        acc = 100 * correct / total
        print(f"Epoch {epoch+1}/{num_epochs} | Loss: {total_loss/len(train_loader):.3f} | Acc: {acc:.2f}%")

    return model

3.5 Batch Normalization — Why It Matters

Batch Normalization (Ioffe & Szegedy, 2015) normalizes the activations within a mini-batch:

x̂ᵢ = (xᵢ - μ_B) / √(σ²_B + ε)
yᵢ = γ·x̂ᵢ + β

Where μ_B and σ²_B are the batch mean and variance, ε is a small constant for numerical stability, and γ, β are learnable scale and shift parameters.

Why it helps: 1. Reduces internal covariate shift — the distribution of activations shifts as earlier layers update, forcing later layers to constantly re-adapt. BN stabilizes these distributions. 2. Acts as regularization (reduces need for dropout) 3. Allows higher learning rates 4. Reduces sensitivity to initialization

During inference, batch statistics are replaced by running averages computed during training (model.eval() mode in PyTorch).

## Chapter 4: Recurrent Neural Networks — Sequential Memory

4.1 Why Sequential Processing?

Many problems are inherently sequential: speech recognition (audio frames over time), text generation (tokens in order), time-series forecasting (measurements at timesteps). Unlike CNNs processing fixed-size windows, RNNs maintain a hidden state — a summary of everything they’ve seen so far.

4.2 The Vanilla RNN

The simplest RNN updates its hidden state at each step:

hₜ = tanh(Wₕₕ·hₜ₋₁ + Wₓₕ·xₜ + b)
yₜ = Wₕᵧ·hₜ + bᵧ

The same weight matrices Wₕₕ, Wₓₕ, Wₕᵧ are applied at every timestep — this is weight sharing across time.

The Problem: Vanilla RNNs cannot learn long-range dependencies. Backpropagation through time (BPTT) requires multiplying gradients through hundreds of timesteps. If |Wₕₕ| < 1, gradients vanish exponentially. If |Wₕₕ| > 1, gradients explode.

4.3 Long Short-Term Memory (LSTM) — Deep Dive

The LSTM (Hochreiter & Schmidhuber, 1997) solves the vanishing gradient problem with a revolutionary idea: a cell state cₜ that can carry information over long distances with minimal transformation.

The cell state is modified by three gates — learned sigmoid-activated filters that decide what to forget, write, and output.

4.3.1 The Four Components

Forget Gate: How much of the old cell state to keep?

fₜ = σ(Wf·[hₜ₋₁, xₜ] + bf)

Input Gate: What new information to write to the cell state?

iₜ = σ(Wᵢ·[hₜ₋₁, xₜ] + bᵢ)
c̃ₜ = tanh(Wc·[hₜ₋₁, xₜ] + bc)   # Candidate cell values

Cell State Update:

cₜ = fₜ ⊙ cₜ₋₁ + iₜ ⊙ c̃ₜ

⊙ is element-wise multiplication. The forget gate erases old info; the input gate writes new info.

Output Gate: What to expose as hidden state?

oₜ = σ(Wo·[hₜ₋₁, xₜ] + bo)
hₜ = oₜ ⊙ tanh(cₜ)

The key insight: gradients can flow through cₜ via the addition operation (gradient of addition is 1), not multiplication — so they don’t vanish! The forget gate can learn to be near 1 for important information, creating near-perfect gradient highways across time.

4.3.2 Full LSTM Implementation

class LSTMCell(nn.Module):
    """
    Single LSTM cell — processes one timestep.
    Implementing from scratch to understand all gate mechanics.
    """
    def __init__(self, input_size, hidden_size):
        super().__init__()
        self.hidden_size = hidden_size

        # All four gates computed in one efficient matrix operation
        # Output size is 4 * hidden_size: [f_gate | i_gate | c_candidate | o_gate]
        self.W_ih = nn.Linear(input_size, 4 * hidden_size)   # Input → gates
        self.W_hh = nn.Linear(hidden_size, 4 * hidden_size)  # Hidden → gates

        self._init_weights()

    def _init_weights(self):
        # Orthogonal initialization for hidden-to-hidden weights
        nn.init.orthogonal_(self.W_hh.weight)
        # Forget gate bias initialized to 1 — makes it remember by default
        nn.init.constant_(self.W_ih.bias[self.hidden_size:2*self.hidden_size], 1.0)
        nn.init.constant_(self.W_hh.bias[self.hidden_size:2*self.hidden_size], 1.0)

    def forward(self, x, h_prev, c_prev):
        """
        x:      [batch, input_size]   — current input
        h_prev: [batch, hidden_size]  — previous hidden state
        c_prev: [batch, hidden_size]  — previous cell state

        Returns:
        h_next: [batch, hidden_size]
        c_next: [batch, hidden_size]
        """
        # Compute all gates in one pass (4x efficient vs. separate matrices)
        gates = self.W_ih(x) + self.W_hh(h_prev)  # [batch, 4*hidden]

        # Split into 4 equal parts
        f_gate, i_gate, c_cand, o_gate = gates.chunk(4, dim=1)
        # Each: [batch, hidden_size]

        # Apply gate activations
        f = torch.sigmoid(f_gate)    # Forget: 0=forget, 1=keep
        i = torch.sigmoid(i_gate)    # Input: 0=ignore, 1=write
        g = torch.tanh(c_cand)       # Candidate values: [-1, 1]
        o = torch.sigmoid(o_gate)    # Output: 0=hide, 1=expose

        # Update cell state: forget old + write new
        c_next = f * c_prev + i * g

        # Compute hidden state: gated tanh of cell state
        h_next = o * torch.tanh(c_next)

        return h_next, c_next


class LSTM(nn.Module):
    """
    Multi-layer LSTM that processes full sequences.
    """
    def __init__(self, input_size, hidden_size, num_layers=1,
                 dropout=0.0, bidirectional=False):
        super().__init__()
        self.hidden_size = hidden_size
        self.num_layers = num_layers
        self.bidirectional = bidirectional
        self.num_directions = 2 if bidirectional else 1

        # Create a cell for each layer and direction
        self.cells = nn.ModuleList()
        for layer in range(num_layers):
            for direction in range(self.num_directions):
                in_size = input_size if layer == 0 else hidden_size * self.num_directions
                self.cells.append(LSTMCell(in_size, hidden_size))

        self.dropout = nn.Dropout(dropout) if dropout > 0 else nn.Identity()

    def forward(self, x, initial_state=None):
        """
        x: [batch, seq_len, input_size]
        Returns:
            output: [batch, seq_len, hidden_size * num_directions]
            (h_n, c_n): final states [num_layers*directions, batch, hidden_size]
        """
        batch_size, seq_len, _ = x.shape
        device = x.device

        # Initialize hidden and cell states to zeros
        if initial_state is None:
            h = torch.zeros(self.num_layers * self.num_directions,
                            batch_size, self.hidden_size, device=device)
            c = torch.zeros_like(h)
        else:
            h, c = initial_state

        layer_input = x  # [batch, seq, input_size]

        h_list, c_list = [], []

        for layer_idx in range(self.num_layers):
            # Forward direction
            fwd_cell = self.cells[layer_idx * self.num_directions]
            fwd_h = h[layer_idx * self.num_directions]
            fwd_c = c[layer_idx * self.num_directions]

            fwd_outputs = []
            for t in range(seq_len):
                fwd_h, fwd_c = fwd_cell(layer_input[:, t, :], fwd_h, fwd_c)
                fwd_outputs.append(fwd_h)

            fwd_seq = torch.stack(fwd_outputs, dim=1)  # [batch, seq, hidden]

            if self.bidirectional:
                # Backward direction: process sequence in reverse
                bwd_cell = self.cells[layer_idx * 2 + 1]
                bwd_h = h[layer_idx * 2 + 1]
                bwd_c = c[layer_idx * 2 + 1]

                bwd_outputs = []
                for t in reversed(range(seq_len)):
                    bwd_h, bwd_c = bwd_cell(layer_input[:, t, :], bwd_h, bwd_c)
                    bwd_outputs.append(bwd_h)
                bwd_outputs.reverse()  # Reorder to match forward direction
                bwd_seq = torch.stack(bwd_outputs, dim=1)

                layer_output = torch.cat([fwd_seq, bwd_seq], dim=-1)
                h_list.extend([fwd_h, bwd_h])
                c_list.extend([fwd_c, bwd_c])
            else:
                layer_output = fwd_seq
                h_list.append(fwd_h)
                c_list.append(fwd_c)

            # Apply dropout between layers (not on final layer)
            if layer_idx < self.num_layers - 1:
                layer_input = self.dropout(layer_output)
            else:
                layer_input = layer_output

        h_n = torch.stack(h_list, dim=0)   # [num_layers*directions, batch, hidden]
        c_n = torch.stack(c_list, dim=0)

        return layer_output, (h_n, c_n)

class LSTMLanguageModel(nn.Module):

        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim)

                            dropout=dropout, batch_first=True)
        self.drop = nn.Dropout(dropout)
        self.decoder = nn.Linear(hidden_size, vocab_size)
        # Tie weights: embedding and decoder share parameters
        # This is a critical regularization technique for language models
        self.decoder.weight = self.embedding.weight

    def forward(self, x, hidden=None):
        """
        x:      [batch, seq_len] — token indices
        Returns logits [batch, seq_len, vocab_size] and new hidden state
        """
        emb = self.drop(self.embedding(x))           # [batch, seq, embed_dim]
        lstm_out, hidden = self.lstm(emb, hidden)    # [batch, seq, hidden]
        lstm_out = self.drop(lstm_out)
        logits = self.decoder(lstm_out)              # [batch, seq, vocab_size]
        return logits, hidden

    def detach_hidden(self, hidden):
        """Detach hidden state to prevent backprop through entire history."""
        return tuple(h.detach() for h in hidden)

4.4 Gated Recurrent Unit (GRU) — Streamlined Gating

The GRU (Cho et al., 2014) is a simplification of the LSTM. It merges the forget and input gates into a single update gate and eliminates the separate cell state.

Update Gate: Controls how much of the previous hidden state to keep.

zₜ = σ(Wz·[hₜ₋₁, xₜ])

Reset Gate: Controls how much of the previous hidden state influences the candidate.

rₜ = σ(Wr·[hₜ₋₁, xₜ])

Candidate Hidden State:

h̃ₜ = tanh(W·[rₜ⊙hₜ₋₁, xₜ])

Final Update:

hₜ = (1 - zₜ) ⊙ hₜ₋₁ + zₜ ⊙ h̃ₜ

When zₜ ≈ 0: the hidden state is almost unchanged (the unit forgets nothing — pure memory). When zₜ ≈ 1: the hidden state is completely replaced by the new candidate.

class GRUCell(nn.Module):
    """
    Single GRU cell. Fewer parameters than LSTM, often comparable performance.
    """
    def __init__(self, input_size, hidden_size):
        super().__init__()
        self.hidden_size = hidden_size

        # Update and reset gates
        self.W_z = nn.Linear(input_size + hidden_size, hidden_size)
        self.W_r = nn.Linear(input_size + hidden_size, hidden_size)
        # Candidate hidden state
        self.W_h = nn.Linear(input_size + hidden_size, hidden_size)

    def forward(self, x, h_prev):
        """
        x:      [batch, input_size]
        h_prev: [batch, hidden_size]
        Returns: h_next [batch, hidden_size]
        """
        combined = torch.cat([x, h_prev], dim=1)   # [batch, input+hidden]

        # Compute gates
        z = torch.sigmoid(self.W_z(combined))      # Update gate
        r = torch.sigmoid(self.W_r(combined))      # Reset gate

        # Candidate: reset gate modulates how much past state to use
        combined_r = torch.cat([x, r * h_prev], dim=1)
        h_candidate = torch.tanh(self.W_h(combined_r))

        # Final hidden state: interpolate between old and new
        h_next = (1 - z) * h_prev + z * h_candidate

        return h_next


class GRUSequenceModel(nn.Module):
    """
    GRU-based model for sequence classification (e.g., sentiment analysis).
    """
    def __init__(self, vocab_size, embed_dim, hidden_size,
                 num_layers, num_classes, dropout=0.3):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        self.gru = nn.GRU(embed_dim, hidden_size, num_layers,
                          batch_first=True, dropout=dropout,
                          bidirectional=True)
        # Attention over all hidden states (instead of just final state)
        self.attention = nn.Linear(hidden_size * 2, 1)
        self.classifier = nn.Sequential(
            nn.Dropout(dropout),
            nn.Linear(hidden_size * 2, hidden_size),
            nn.ReLU(),
            nn.Dropout(dropout),
            nn.Linear(hidden_size, num_classes)
        )

    def forward(self, x, lengths=None):
        """
        x:       [batch, seq_len] — padded token ids
        lengths: [batch] — actual sequence lengths (for packing)
        """
        emb = self.embedding(x)           # [batch, seq, embed]

        # Pack padded sequences for efficiency (skip padding in GRU)
        if lengths is not None:
            emb = nn.utils.rnn.pack_padded_sequence(
                emb, lengths.cpu(), batch_first=True, enforce_sorted=False
            )

        gru_out, _ = self.gru(emb)        # [batch, seq, 2*hidden]

        if lengths is not None:
            gru_out, _ = nn.utils.rnn.pad_packed_sequence(gru_out, batch_first=True)

        # Attention mechanism over sequence dimension
        attn_scores = self.attention(gru_out).squeeze(-1)  # [batch, seq]
        if lengths is not None:
            # Mask padding positions
            for i, l in enumerate(lengths):
                attn_scores[i, l:] = float('-inf')
        attn_weights = F.softmax(attn_scores, dim=1).unsqueeze(-1)  # [batch, seq, 1]
        context = (gru_out * attn_weights).sum(dim=1)               # [batch, 2*hidden]

        return self.classifier(context)   # [batch, num_classes]

## Chapter 5: Generative Adversarial Networks — Learning to Create

5.1 The Core Idea: A Brilliant Metaphor

A GAN (Goodfellow et al., 2014) involves two networks locked in a competitive game:

  • The Generator (G): A forger who creates fake artwork. It takes random noise z as input and outputs synthetic data (images, audio, text).

  • The Discriminator (D): An art critic who tries to distinguish real artwork from the forger’s fakes. It takes an input sample and outputs a probability that it’s real.

They are trained adversarially: - G tries to fool D by generating more realistic fakes - D tries to get better at detecting fakes

The remarkable result: through this adversarial pressure, G learns to generate incredibly realistic data — without ever directly seeing the training data distribution. G only knows whether D was fooled.

5.2 The Adversarial Loss — Formal Derivation

The GAN minimax game is formalized as:

min_G max_D V(D, G) = E_{x~p_data}[log D(x)] + E_{z~p_z}[log(1 - D(G(z)))]

Breaking this down: - E_{x~p_data}[log D(x)]: Discriminator should output high probability (close to 1) for real data x. log D(x) is maximized when D(x) → 1. - E_{z~p_z}[log(1 - D(G(z)))]: Discriminator should output low probability for fake data G(z). log(1 - D(G(z))) is maximized when D(G(z)) → 0.

Discriminator wants to maximize this (correctly classify real and fake). Generator wants to minimize this (fool the discriminator into classifying fakes as real).

The non-saturating trick: In practice, instead of minimizing log(1 - D(G(z))), the generator minimizes -log(D(G(z))) (equivalent to maximizing log(D(G(z)))). This provides stronger gradients early in training when the generator is terrible and the discriminator easily rejects its outputs.

5.3 Building Generator and Discriminator Networks

5.3.1 DCGAN — Deep Convolutional GAN

DCGAN (Radford et al., 2015) established the architectural guidelines that made GANs stable to train:

Generator (Decoder-like): - Takes latent vector z and upsamples to image resolution - Uses transposed convolutions (fractionally strided convolutions) for upsampling - BatchNorm on all layers except output - ReLU activations, tanh on output layer

Discriminator (Encoder-like): - Takes image and downsamples to scalar - Uses strided convolutions instead of pooling - BatchNorm on all layers except input - LeakyReLU activations, sigmoid on output

class DCGANGenerator(nn.Module):
    """
    Generator for 64×64 images.
    Architecture: z [100] → linear → reshape → 4 transposed conv blocks → 64×64 RGB
    """
    def __init__(self, latent_dim=100, img_channels=3, feature_maps=64):
        super().__init__()
        self.latent_dim = latent_dim
        ngf = feature_maps  # Base number of feature maps

        # Project and reshape noise vector
        self.project = nn.Sequential(
            nn.Linear(latent_dim, ngf * 8 * 4 * 4),
            nn.Unflatten(1, (ngf * 8, 4, 4))
        )
        # Note: Unflatten reshapes to [batch, ngf*8, 4, 4]

        # Transposed convolution blocks: each doubles spatial size
        # 4×4 → 8×8 → 16×16 → 32×32 → 64×64
        self.blocks = nn.Sequential(
            # Block 1: 4×4 → 8×8
            self._tconv_block(ngf * 8, ngf * 4),
            # Block 2: 8×8 → 16×16
            self._tconv_block(ngf * 4, ngf * 2),
            # Block 3: 16×16 → 32×32
            self._tconv_block(ngf * 2, ngf),
            # Block 4: 32×32 → 64×64 (output layer — no BN, tanh activation)
            nn.ConvTranspose2d(ngf, img_channels,
                               kernel_size=4, stride=2, padding=1, bias=False),
            nn.Tanh()  # Output in [-1, 1] (match normalized images)
        )

    def _tconv_block(self, in_ch, out_ch):
        """Transposed conv block: upsample by 2×."""
        return nn.Sequential(
            nn.ConvTranspose2d(in_ch, out_ch,
                               kernel_size=4, stride=2, padding=1, bias=False),
            nn.BatchNorm2d(out_ch),
            nn.ReLU(inplace=True)
        )

    def forward(self, z):
        """z: [batch, latent_dim]"""
        x = self.project(z)    # [batch, ngf*8, 4, 4]
        return self.blocks(x)  # [batch, 3, 64, 64]


class DCGANDiscriminator(nn.Module):
    """
    Discriminator for 64×64 images.
    Architecture: 64×64 RGB → 4 conv blocks → linear → scalar
    """
    def __init__(self, img_channels=3, feature_maps=64):
        super().__init__()
        ndf = feature_maps
        # No BatchNorm on first layer (input is raw pixels)
        self.model = nn.Sequential(
            # Block 1: 64×64 → 32×32
            nn.Conv2d(img_channels, ndf, kernel_size=4, stride=2, padding=1, bias=False),
            nn.LeakyReLU(0.2, inplace=True),  # LeakyReLU prevents dead neurons

            # Block 2: 32×32 → 16×16
            nn.Conv2d(ndf, ndf * 2, kernel_size=4, stride=2, padding=1, bias=False),
            nn.BatchNorm2d(ndf * 2),
            nn.LeakyReLU(0.2, inplace=True),

            # Block 3: 16×16 → 8×8
            nn.Conv2d(ndf * 2, ndf * 4, kernel_size=4, stride=2, padding=1, bias=False),
            nn.BatchNorm2d(ndf * 4),
            nn.LeakyReLU(0.2, inplace=True),

            # Block 4: 8×8 → 4×4
            nn.Conv2d(ndf * 4, ndf * 8, kernel_size=4, stride=2, padding=1, bias=False),
            nn.BatchNorm2d(ndf * 8),
            nn.LeakyReLU(0.2, inplace=True),

            # Output: 4×4 → 1×1 → scalar
            nn.Conv2d(ndf * 8, 1, kernel_size=4, stride=1, padding=0, bias=False),
            nn.Flatten(),
            nn.Sigmoid()  # Output probability: real (1) or fake (0)
        )

    def forward(self, x):
        """x: [batch, 3, 64, 64]"""
        return self.model(x).squeeze()  # [batch]

5.4 The Full GAN Training Loop

GAN training is a two-player game — we alternate between optimizing the discriminator and the generator.

def weights_init(m):
    """DCGAN weight initialization: Conv weights from N(0, 0.02)"""
    classname = m.__class__.__name__
    if classname.find('Conv') != -1:
        nn.init.normal_(m.weight.data, 0.0, 0.02)
    elif classname.find('BatchNorm') != -1:
        nn.init.normal_(m.weight.data, 1.0, 0.02)
        nn.init.constant_(m.bias.data, 0)


def train_gan(
    generator, discriminator, dataloader,
    latent_dim=100, num_epochs=200, lr=0.0002,
    beta1=0.5,  # Adam beta1: lower than default for GANs
    device='cuda'
):
    """
    Full DCGAN training loop with all the stability tricks.
    """
    generator = generator.to(device)
    discriminator = discriminator.to(device)

    # Apply DCGAN weight initialization
    generator.apply(weights_init)
    discriminator.apply(weights_init)

    # Binary cross-entropy loss for adversarial training
    criterion = nn.BCELoss()

    # Separate optimizers for G and D
    # Key: lower beta1=0.5 (instead of default 0.9) reduces momentum, helps stability
    opt_G = torch.optim.Adam(generator.parameters(),
                             lr=lr, betas=(beta1, 0.999))
    opt_D = torch.optim.Adam(discriminator.parameters(),
                             lr=lr, betas=(beta1, 0.999))

    # Fixed noise for monitoring training progress
    fixed_noise = torch.randn(64, latent_dim, device=device)

    # Label conventions
    real_label = 1.0
    fake_label = 0.0

    # Tracking metrics
    G_losses, D_losses = [], []

    print("Starting GAN Training...")
    for epoch in range(num_epochs):
        for batch_idx, (real_images, _) in enumerate(dataloader):
            batch_size = real_images.size(0)
            real_images = real_images.to(device)

            # ==========================================
            # STEP 1: Train the Discriminator
            # Goal: maximize log(D(x)) + log(1 - D(G(z)))
            # ==========================================
            discriminator.zero_grad()

            # --- Train on real batch ---
            real_labels = torch.full((batch_size,), real_label, device=device)
            # Forward pass real images through D
            pred_real = discriminator(real_images)
            # Loss: D should output 1 for real images
            loss_D_real = criterion(pred_real, real_labels)
            loss_D_real.backward()

            # --- Train on fake batch ---
            # Generate fake images
            noise = torch.randn(batch_size, latent_dim, device=device)
            fake_images = generator(noise)

            fake_labels = torch.full((batch_size,), fake_label, device=device)
            # Note: .detach() — don't backprop through G when training D
            pred_fake = discriminator(fake_images.detach())
            # Loss: D should output 0 for fake images
            loss_D_fake = criterion(pred_fake, fake_labels)
            loss_D_fake.backward()

            # Total discriminator loss
            loss_D = loss_D_real + loss_D_fake
            opt_D.step()

            # ==========================================
            # STEP 2: Train the Generator
            # Goal: maximize log(D(G(z))) [non-saturating]
            # ==========================================
            generator.zero_grad()

            # Generate new fake images (or reuse — but new is more stable)
            noise = torch.randn(batch_size, latent_dim, device=device)
            fake_images = generator(noise)

            # For generator training, we WANT D to think these are real
            # So we use real_label=1 as target — generator succeeds when D(G(z))=1
            pred_G = discriminator(fake_images)  # No .detach() — we want gradients through G
            loss_G = criterion(pred_G, real_labels)  # "Fool D into saying these are real"
            loss_G.backward()
            opt_G.step()

            # Logging
            G_losses.append(loss_G.item())
            D_losses.append(loss_D.item())

        if (epoch + 1) % 10 == 0:
            print(f"Epoch [{epoch+1}/{num_epochs}] "
                  f"Loss_D: {loss_D.item():.4f}  "
                  f"Loss_G: {loss_G.item():.4f}  "
                  f"D(x): {pred_real.mean().item():.3f}  "
                  f"D(G(z)): {pred_fake.mean().item():.3f}/{pred_G.mean().item():.3f}")

    return G_losses, D_losses

5.5 GAN Training Instability: The Deep Problems and Solutions

GANs are notoriously hard to train. Three fundamental problems arise:

5.5.1 Mode Collapse

The generator learns to produce a small variety of samples (or even just one) that consistently fool the discriminator, ignoring the full data distribution.

Symptoms: All generated faces look similar; generated digits all look like “1”.

Solutions: - Minibatch discrimination: Discriminator compares samples within a mini-batch, penalizing when all fake images are similar. - Unrolled GANs: Generator optimizes against future discriminator states. - Wasserstein GAN (WGAN): Replaces JS divergence with Wasserstein distance — more stable gradient signal.

5.5.2 Vanishing Gradients

When the discriminator becomes too good, it outputs near-0 for all fakes, and log(1 - D(G(z))) ≈ log(1) = 0 — zero gradient for the generator.

Solution: WGAN, non-saturating loss, gradient penalty.

5.5.3 Training Imbalance

If D or G is much stronger than the other, training breaks down.

Solution: Train D multiple times per G update, or vice versa. Two-timescale update rules (TTUR): different learning rates.

5.6 Wasserstein GAN (WGAN-GP) — State-of-the-Art Stability

WGAN replaces the discriminator with a critic that outputs an unrestricted real number (not a probability). The loss becomes:

Loss_Critic = E[Critic(fake)] - E[Critic(real)]   ← Critic minimizes
Loss_G      = -E[Critic(G(z))]                    ← Generator maximizes critic score

The 1-Lipschitz constraint on the critic (required by Wasserstein theory) is enforced via gradient penalty:

GP = E[(||∇_x̂ Critic(x̂)||₂ - 1)²]

Where x̂ is a random interpolation between real and fake: x̂ = ε·x_real + (1-ε)·x_fake.

class WGANGPCritic(nn.Module):
    """
    WGAN-GP Critic (discriminator without sigmoid — outputs raw score).
    No BatchNorm! (incompatible with gradient penalty). Use LayerNorm or SpectralNorm.
    """
    def __init__(self, img_channels=3, feature_maps=64):
        super().__init__()
        ndf = feature_maps
        self.model = nn.Sequential(
            # No BN in critic for WGAN-GP
            nn.Conv2d(img_channels, ndf, 4, 2, 1),
            nn.LeakyReLU(0.2, inplace=True),

            nn.Conv2d(ndf, ndf*2, 4, 2, 1),
            nn.InstanceNorm2d(ndf*2, affine=True),  # InstanceNorm instead of BatchNorm
            nn.LeakyReLU(0.2, inplace=True),

            nn.Conv2d(ndf*2, ndf*4, 4, 2, 1),
            nn.InstanceNorm2d(ndf*4, affine=True),
            nn.LeakyReLU(0.2, inplace=True),

            nn.Conv2d(ndf*4, ndf*8, 4, 2, 1),
            nn.InstanceNorm2d(ndf*8, affine=True),
            nn.LeakyReLU(0.2, inplace=True),

            nn.Conv2d(ndf*8, 1, 4, 1, 0),
            nn.Flatten()
            # NO sigmoid! Critic outputs raw real number
        )

    def forward(self, x):
        return self.model(x).squeeze()


def gradient_penalty(critic, real_images, fake_images, device):
    """
    Compute WGAN-GP gradient penalty.
    Enforces 1-Lipschitz constraint by penalizing gradient norm.
    """
    batch_size = real_images.size(0)

    # Random interpolation coefficient: ε ~ Uniform[0,1]
    epsilon = torch.rand(batch_size, 1, 1, 1, device=device)

    # Interpolated samples: x̂ = ε·real + (1-ε)·fake
    interpolated = epsilon * real_images + (1 - epsilon) * fake_images.detach()
    interpolated.requires_grad_(True)

    # Critic score for interpolated samples
    critic_score = critic(interpolated)

    # Compute gradients of critic scores w.r.t. interpolated samples
    gradients = torch.autograd.grad(
        outputs=critic_score,
        inputs=interpolated,
        grad_outputs=torch.ones_like(critic_score),
        create_graph=True,   # Need second-order gradients for optimization
        retain_graph=True,
    )[0]

    # Gradient norm: flatten spatial dims, compute L2 norm per sample
    gradients = gradients.view(batch_size, -1)
    gradient_norm = gradients.norm(2, dim=1)           # [batch]

    # Penalty: (||∇||₂ - 1)² — penalize deviations from unit gradient norm
    penalty = ((gradient_norm - 1) ** 2).mean()

    return penalty


def train_wgan_gp(generator, critic, dataloader,
                  latent_dim=100, num_epochs=100,
                  n_critic=5,    # Train critic n_critic times per generator update
                  lambda_gp=10,  # GP weight
                  lr=1e-4, device='cuda'):
    """
    WGAN-GP training loop.
    """
    generator, critic = generator.to(device), critic.to(device)

    # Note: WGAN uses Adam with lower learning rate and beta1=0
    opt_G = torch.optim.Adam(generator.parameters(), lr=lr, betas=(0.0, 0.9))
    opt_C = torch.optim.Adam(critic.parameters(),    lr=lr, betas=(0.0, 0.9))

    for epoch in range(num_epochs):
        for real_images, _ in dataloader:
            batch_size = real_images.size(0)
            real_images = real_images.to(device)

            # === Train Critic (n_critic times per G update) ===
            for _ in range(n_critic):
                noise = torch.randn(batch_size, latent_dim, device=device)
                fake_images = generator(noise).detach()

                # Wasserstein loss: maximize E[C(real)] - E[C(fake)]
                loss_C = (critic(fake_images).mean()   # C wants fake scores LOW
                         - critic(real_images).mean()  # C wants real scores HIGH
                         + lambda_gp * gradient_penalty(critic, real_images, fake_images, device))

                opt_C.zero_grad()
                loss_C.backward()
                opt_C.step()

            # === Train Generator ===
            noise = torch.randn(batch_size, latent_dim, device=device)
            fake_images = generator(noise)

            # G wants critic to score its fakes as HIGH as possible
            loss_G = -critic(fake_images).mean()

            opt_G.zero_grad()
            loss_G.backward()
            opt_G.step()

        if (epoch + 1) % 10 == 0:
            print(f"Epoch {epoch+1}: Loss_C={loss_C.item():.4f} | Loss_G={loss_G.item():.4f}")

5.7 Conditional GANs (cGAN)

Standard GANs generate random samples — we can’t control what class or attribute is generated. Conditional GANs (Mirza & Osindero, 2014) condition both G and D on additional information y (a class label, text description, or reference image).

class ConditionalGenerator(nn.Module):
    """Generator conditioned on a class label."""
    def __init__(self, latent_dim, num_classes, embed_dim, img_size):
        super().__init__()
        self.label_emb = nn.Embedding(num_classes, embed_dim)
        in_dim = latent_dim + embed_dim

        self.model = nn.Sequential(
            nn.Linear(in_dim, 256),
            nn.LeakyReLU(0.2),
            nn.BatchNorm1d(256),
            nn.Linear(256, 512),
            nn.LeakyReLU(0.2),
            nn.BatchNorm1d(512),
            nn.Linear(512, img_size * img_size),
            nn.Tanh()
        )
        self.img_size = img_size

    def forward(self, noise, labels):
        """
        noise:  [batch, latent_dim]
        labels: [batch] — class indices
        """
        label_embed = self.label_emb(labels)           # [batch, embed_dim]
        x = torch.cat([noise, label_embed], dim=1)    # [batch, latent+embed]
        img = self.model(x)
        return img.view(img.size(0), 1, self.img_size, self.img_size)

5.9 Evaluating GANs

Unlike supervised learning, we can’t use simple accuracy. GAN evaluation is an open research problem. Key metrics:

Fréchet Inception Distance (FID): The most widely used metric. Computes the Fréchet distance between the distributions of Inception features from real and generated images. Lower is better.

# Conceptual FID calculation (uses scipy in practice)
def compute_fid(mu_real, sigma_real, mu_fake, sigma_fake):
    """
    FID = ||μ_r - μ_g||² + Tr(Σ_r + Σ_g - 2(Σ_r·Σ_g)^(1/2))
    """
    from scipy.linalg import sqrtm
    import numpy as np
    diff = mu_real - mu_fake
    covmean = sqrtm(sigma_real @ sigma_fake)
    if np.iscomplexobj(covmean):
        covmean = covmean.real
    fid = (diff @ diff + np.trace(sigma_real + sigma_fake - 2 * covmean))
    return fid

Inception Score (IS): Measures both quality (sharp predictions for single class) and diversity (uniform marginal distribution across classes). Higher is better.

Precision and Recall: Precision measures sample quality; recall measures coverage of the real data manifold.

Chapter 6: Putting It All Together — The Unified View

6.1 Architecture Design Principles

After diving deep into five paradigms, we can identify universal principles:

  1. Residuals everywhere: Skip connections solve vanishing gradients and ease optimization. Used in Transformers (residual connections), ResNets, and modern RNNs.

  2. Normalization is essential: Layer Norm (Transformer), Batch Norm (CNNs), Instance Norm (GANs) — each architecture uses the right normalization for its structure.

  3. Attention is general aggregation: Multi-head attention in Transformers, graph attention in GATs, temporal attention in GRUs — all are forms of learned weighted aggregation.

  4. Scale matters: More parameters, more data, and more compute consistently improve performance across all architectures (the “bitter lesson”).

  5. Inductive biases: Good architectures encode assumptions about data structure. CNNs assume spatial locality. RNNs assume temporal order. GNNs assume relational structure. Transformers make fewer assumptions (more general, but need more data).

6.2 The Connections Between Architectures

  • Transformer = Fully-connected GNN: Self-attention is message passing where every node can attend to every other node. Transformers are GNNs on a complete graph with learned edge weights.

  • GNN on grid = CNN: When the graph is a regular 2D grid with local connections, graph convolution reduces to standard convolution.

  • Attention in RNNs: The Bahdanau attention mechanism (precursor to Transformers) is attention added on top of encoder-decoder RNNs.

  • GAN generator = Decoder: The Transformer decoder, CNN decoder (transposed convolutions), and GAN generator all take compact representations and expand them into rich outputs.
